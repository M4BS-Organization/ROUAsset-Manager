# ドキュメント調査: CHUKI/IDOLST JOKEN ブラックボックステスト

## 1. VB.NET WinForms でのブラックボックステスト手法

- **同値分割法**: 入力データを同等クラスに分割し代表値でテスト
- **境界値分析**: 境界値・境界値±1をテスト。同値分割と組み合わせ
- **実際のユーザーフローに基づくテスト**: 業務フロー全体を通したテスト

## 2. VB.NET コンソールアプリによるDLL呼び出しテストパターン

- クラスライブラリ（DLL）をコンソールアプリから参照してテスト
- 名前空間をImportsして公開メソッドを直接呼び出し
- 結果をコンソール出力（PASS/FAIL）

## 3. Access VBA → VB.NET 移行時のテスト戦略

- **Convert-Test-Fix方式**: コード変換→テスト→修正のサイクル
- **入出力トレース比較**: 同じ入力を与えてステップバイステップで出力比較
- **E2Eユーザー受入テスト**: レガシーと同一動作を確認

## 4. リース会計における中間期計算・異動リスト条件

### IFRS16/新リース会計基準の中間期処理
- リース負債の再測定: 見積り変更時に再測定が必要
- 契約変更: 使用権追加＋対価が独立価格に見合う場合は独立リース
- 中間期ではリース期間・支払額・割引率の変更が使用権資産の再計算に直結

### 異動リスト条件のロジック
- 新規取得、解約、条件変更、移管等をリスト化→仕訳処理生成
- 新リース会計基準では仕訳量が従来の3〜5倍

### LeaseM4BSシステムとの関連
- f_CHUKI_JOKEN: 中間期における条件判定ロジック（再計算・再見積り条件判定）
- f_IDOLST_JOKEN: 異動（新規・変更・解約等）の仕訳出力対象条件判定

## 5. テスト設計への示唆

1. **コンソールアプリテストパターン**: DataAccess DLLを参照するコンソールアプリ
2. **同値分割 + 境界値分析**: リース期間、金額、日付等の入力パラメータ
3. **Access版との入出力比較**: Convert-Test-Fix方式
4. **条件分岐の網羅**: 各分岐パスをテストケースでカバー
