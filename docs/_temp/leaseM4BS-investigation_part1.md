## システム概要

### リースM4BSとは

リースM4BS（Lease Management for Business System）は、イルテックス社が開発したリース管理システムである。IFRSおよび日本基準に対応したオペレーティングリース・ファイナンスリースの契約管理・会計仕訳出力を業務範囲とする。

主な機能は以下のとおり。

- リース契約書の登録・管理（契約書ヘッダ・物件明細）
- 月次仕訳計上（計上仕訳・支払仕訳・経費仕訳）
- 返済スケジュール・償却スケジュールの自動計算
- 仕訳の外部システム（会計パッケージ）への出力
- 各種帳票・一覧表の出力
- マスタデータ（企業・口座・仕訳科目等）の保守

### 技術スタック

| 世代 | 実行環境 | 主要技術 | データベース |
|------|---------|---------|------------|
| Access版（現行） | Microsoft Access 2016 64bit Runtime | VBA / Access Form | Access MDB / Oracle |
| VB.NET版（移行中） | .NET Framework 4.7.2 | WinForms / VB.NET | PostgreSQL |

### 共有資料の構成

本調査は以下の3種類の資料をもとに実施した。

| AccessVBA | VBAモジュール群（約373モジュール） |

### GUI定義ファイルの構造

AccessGUIフォームの定義ファイルは以下の構造を持つ。

```
Access Form定義言語 Version 21

Begin Form
    ...（フォームプロパティ）
    RecordSource = "tw_XXXX"     ' データソース（tw_プレフィックスのテーブル/ビュー）
    Begin Section
        ...（セクション定義）
        Begin Control
            ...（コントロール定義）
            OnOpen      = "[Event Procedure]"   ' VBAイベントハンドラ連携
            OnClose     = "[Event Procedure]"
        End
    End
    NameMap                       ' コントロール名のUnicode/16進エンコード
    PrtDevMode                    ' プリンター設定（16進数バイナリ）
End
```

主な特徴は以下のとおり。

- `RecordSource` にはプレフィックス `tw_` のテーブルまたはビューを指定
- イベントハンドラ（`OnOpen`、`OnClose`、`OnClick` 等）は `[Event Procedure]` として宣言し、VBAモジュールと連携
- `NameMap` セクションでコントロール名をUnicodeまたは16進数でエンコード
- `PrtDevMode` セクションに16進数バイナリ形式のプリンター設定を格納

## システム起動フローとメニュー構成

### 起動処理の流れ

システム起動時に `p_StartUp_gMain()` が呼び出され、以下の順序で初期化処理が実行される。

```
p_StartUp_gMain()
  ├─ Accessバージョン確認（2000～2016対応）
  ├─ 和暦配列 tgEras() 初期化（明治・大正・昭和・平成・令和）
  ├─ インストールフォルダチェック（ProgramFiles直下インストール時に警告）
  ├─ ACCESSウィンドウ最大化
  ├─ ワークDB初期化
  │   ├─ LM4BSwork.mdb       （ワーク用）
  │   ├─ LM4BSmdld.mdb       （モデル用）
  │   └─ LM4BSImpWkDB.mdb    （Excel取込用）
  ├─ カスタマイズ設定読込（T_Customize テーブル → igCUSTM_TYPE に格納）
  ├─ メニューバー生成（tcon_MenuMatrix テーブル → 5階層メニュー構築）
  ├─ セキュリティ設定（権限グループ別メニュー表示/非表示制御）
  └─ ログ初期化（Tempフォルダに LM42log.mdb を生成）
```

### ログイン方式

システムはデータベース接続方式によって2種類のログインモードを持つ。

| ログイン方式 | 接続先 | 概要 |
|-----------|-------|------|
| JETログイン | Access MDB | 標準構成。ローカルまたはファイルサーバー上のMDB直接接続 |
| Oracleログイン | Oracle DB | `T_Customize` の「サーバーDBはOracle」フラグが有効な場合に使用 |

ログイン情報は `typ_gLogin` 構造体で管理され、以下のフィールドを保持する。

| フィールド | 内容 |
|-----------|------|
| `UserCD` | ユーザーコード |
| `PWD` | パスワード（暗号化済み） |
| `KngnCd` | 権限グループコード |
| `boAdmin` | 管理者フラグ |
| `boMaster` | マスタ管理フラグ |
| `boAPPROVAL` | 承認フラグ |
| `boLOG` | ログ参照フラグ |
| `boFOutKNGN` | 帳票出力権限フラグ |
| `boPriKNGN` | 印刷権限フラグ |

ログイン試行は最大5回（`cn_gLOGIN_ATTEMPTS_SYS = 5`）に制限されている。

### メニュー構成（5階層、tcon_MenuMatrix）

メニューは `tcon_MenuMatrix` テーブルの定義に基づき、起動時に動的に生成される。最大5階層の階層メニューを構成し、ユーザーの権限グループ（`mMENUBAR_ENABLE_CTL`）によって表示・非表示が制御される。

メニューの大区分は以下のカテゴリで構成されている（フォーム名から推定）。

| カテゴリ | 主な機能 |
|---------|---------|
| 契約管理 | 契約書登録・参照・一覧、解約処理 |
| 物件管理 | 物件登録・参照・変更 |
| 月次処理 | 仕訳計上、支払処理 |
| 仕訳出力 | 計上仕訳・支払仕訳・経費仕訳の外部出力 |
| 帳票出力 | 棚卸・リース料・残高・債務等の帳票 |
| データ取込 | Excelインポート、各種データ取込 |
| マスタ管理 | 各種マスタの保守 |
| ツール | 祝日・消費税・契約番号シーケンス等 |
| システム | ユーザー管理・権限設定・ログ参照 |

### 画面遷移の全体像

起動後の基本的な画面遷移は以下のパターンに従う。

```
[起動]
  └─ p_StartUp_gMain()
       └─ ログイン画面
            └─ メインメニュー（tcon_MenuMatrix 生成）
                 ├─ 条件入力画面（f_*_JOKEN）
                 │    └─ 一覧画面（f_flx_*）
                 │         └─ 明細入力画面（f_KYKH / f_KYKM 等）
                 ├─ 仕訳出力画面（f_仕訳出力標準_* / fc_*）
                 ├─ マスタ一覧（f_flx_M_*）
                 │    └─ マスタ入力（f_M_*_INP）
                 └─ システム設定（0F_SYSTEM等）
```

`f_KIRIKAE`（画面切替フォーム）がモーダルダイアログとして機能し、進捗表示には `f_StatusMeter` が使用される。
