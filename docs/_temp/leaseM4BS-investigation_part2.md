## H2-5: テーブル体系

リースM4BSのテーブルはプレフィックスによって役割が明確に分類されている。以下にプレフィックス別の体系を示す。

### テーブル命名規則

| プレフィックス | 分類 | 用途 |
|-------------|------|------|
| `D_` | 業務データ | 契約・物件・変更履歴等の業務トランザクションデータ |
| `M_` | マスタ | 会社・科目・パターン等の参照データ |
| `T_` | システム設定 | システム動作を制御する設定値 |
| `tcon_` | 設定コントロール | メニュー・MDBパス・メッセージ等のシステム構成 |
| `L_` | ログ | 操作・更新・バックアップの監査ログ |
| `tw_` | ワーク | 計算中間結果・仕訳作業用の一時テーブル |
| `SEC_` | セキュリティ | ユーザー・権限グループ |
| `fc_` (tw_fc_) | 顧客固有ワーク | 顧客別仕訳出力作業テーブル |

---

### 業務データテーブル（D_ 系）

契約情報・変更履歴・計算結果を保持するコアデータ層。

| テーブル名 | 内容 |
|-----------|------|
| `D_KYKH` | 契約書ヘッダ（KYKH_ID, 契約者名 等） |
| `D_KYKM` | 契約書明細（物件情報） |
| `D_HAIF` | 配分データ（費用の部門配賦） |
| `D_HENL` | 変更履歴（返済スケジュール変更等） |
| `D_GSON` | 減損データ |

---

### マスタテーブル（M_ 系）

業務処理で参照される各種マスタデータ。

| テーブル名 | 内容 |
|-----------|------|
| `M_CORP` | 企業マスタ |
| `M_KKNRI` | 契約管理単位 |
| `M_LCPT` | リース料金パターン |
| `M_SHHO` | 支払方法 |
| `M_GENK` | 原価 |
| `M_BCAT` | 物件分類 |
| `M_HKMK` | 配賦目的 |
| `M_SKMK` | 仕訳科目 |
| `M_BKIND` | 物件種別 |
| `M_KOZA` | 口座 |
| `M_GSHA` | リース会社 |
| `M_MCPT` | 月額料金 |
| `M_HKHO` | 配賦方法 |
| `M_SWPTN` | 仕訳パターン（仕訳出力の勘定科目組み合わせ定義） |

---

### システム設定テーブル（T_ 系 / tcon_ 系）

システムの動作を制御する設定・構成情報。

| テーブル名 | 内容 |
|-----------|------|
| `T_SYSTEM` | システム全体設定値（バージョン・パスワードポリシー等） |
| `T_Customize` | 顧客別カスタマイズフラグ（26顧客対応、詳細はH2-9参照） |
| `tcon_MenuMatrix` | メニュー定義（5階層メニュー構成） |
| `tcon_MDBList` | MDBファイルパス管理 |
| `tcon_Message` | メッセージマスタ（エラー・確認文言） |

---

### ログテーブル（L_ 系）

監査・追跡用のログデータ。

| テーブル名 | 内容 |
|-----------|------|
| `L_SLOG` | 操作ログ（画面開閉・ボタン操作等） |
| `L_ULOG` | 更新ログ（データ変更前後の値） |
| `L_BKLOG` | バックアップログ |

ログDBは Tempフォルダ配下の `LM42log.mdb` に格納される（起動時に初期化）。

---

### ワークテーブル（tw_ 系）

計算処理・仕訳出力の中間作業領域。処理完了後に破棄またはクリアされる。

| テーブル名 | 内容 |
|-----------|------|

| `tw_S_TOUGETSU_JOKEN` | 支払仕訳条件ワーク |
| `tw_S_KEIJO_JOKEN` | 計上仕訳条件ワーク |
| `tw_fc_支払仕訳` | 顧客別支払仕訳作業テーブル |
| `tw_fc_計上仕訳` | 顧客別計上仕訳作業テーブル |
| `tw_KITOKU_CMSW2WRK` | 伝票ワーク（固定長テキスト出力用） |
| `tw_KITOKU_APGDHWRK` | 金額概要ワーク |
| `tw_KITOKU_APGDDWRK` | 金額詳細ワーク |
| `tw_KITOKU_APGDSWRK` | 支払ワーク |

ワークテーブルは `LM4BSwork.mdb`（業務ワーク）・`LM4BSmdld.mdb`（モデル）・`LM4BSImpWkDB.mdb`（Excel取込）の3つの別MDBに分散して管理される。

---

### セキュリティテーブル（SEC_ 系）

| テーブル名 | 内容 |
|-----------|------|
| `SEC_USER` | ユーザーアカウント（UserCD, パスワードハッシュ, 権限グループCD等） |

権限グループ・アクセス種別はテーブルではなくメモリ上の構造体（`typ_gLogin`）で管理される（詳細はH2-8参照）。

## H2-6: リース計算ロジック

リースM4BSの計算エンジンはAccessVBA上に実装されており、IFRS16号・リース会計基準に対応した月次計上計算を行う。

### 月次仕訳計上フロー

月次計上処理のエントリポイントは `pc_月次仕訳計上` であり、以下の順で処理が実行される。

```
pc_月次仕訳計上
└─ g月次_v計_契約書to料金TBL_計上()
    ├─ 対象期間の契約データ抽出（D_KYKH / D_KYKM）
    ├─ 契約ループ（各契約ごとに以下を実行）
    │  ├─ m月次_v計_SQLMAKE()       SQL生成（抽出条件組み立て）
    │  ├─ m月次_v計()               利息・償却の統合計算
    │  │  ├─ m月次_v計_SUB_償却関連()  償却関連サブ処理
    │  │  ├─ mMake償却_SCH()           償却スケジュール生成
    │  │  └─ gMake返済_SCH()           返済スケジュール生成
    │  └─ mMakeFUSAI_SCH()         債務スケジュール生成
    ├─ m利用_結果テーブル入力()     計算結果をtwテーブルへ書込
    └─ m登録処理()                  業務テーブルへ確定登録
```

---

### 年金現価計算

仕訳計上の基礎となるリース料の現在価値は年金現価公式で算出される。

- 計算定義: `0f_MNT_tcon_年金現価の計算式`
- 用途: リース開始時の資産計上額（使用権資産）および負債額（リース負債）の算定

```
年金現価 = 定期支払額 × Σ [1 / (1 + r)^n]
           （r = 月利率, n = 残存期間月数）
```

追加借入利子率（仮利率）は `f_T_KARI_RITU` 画面で管理し、`T_SYSTEM` に保存される。

---

### 返済スケジュール生成（gMake返済_SCH）

各リース契約について、リース期間全体の返済計画を生成する。

| 列 | 内容 |
|----|------|
| 期日 | 各返済日（月次） |
| リース料 | 毎月の支払総額 |
| 元本返済額 | 負債元本の減少分 |
| 利息額 | 利息 = 期首残高 × 利率 / 12 |
| 期末残高 | 期首残高 - 元本返済額 |

```
利息 = リース価額 × 利率 / 12 × 月数
```

---

### 償却計算（定額法 / 利息法）

`mMake償却_SCH` が以下2方式を切り替えて処理する。

| 方式 | 概要 | 適用場面 |
|------|------|---------|
| 定額法 | 資産残高 ÷ リース期間（月数）を毎月均等計上 | オペレーティングリース相当 |
| 利息法 | 期首帳簿価額 × 実効利子率で利息を計算し差額を元本返済 | ファイナンスリース（IFRS16対応） |

償却スケジュール結果は `tw_S_CHUKI_KEIJO`（注記計上）・`tw_D_HENL_KEIJO`（変更時）・`tw_D_GSON_KEIJO`（減損時）に格納される。

---

### 債務スケジュール生成（mMakeFUSAI_SCH）

リース負債の長短振替を含む債務スケジュールを生成する。

- **長期負債**: 決算日から1年超のリース負債残高
- **短期負債（流動負債）**: 決算日から1年以内に弁済予定の元本分
- 長短振替タイミングは会計年度終了月を基準に自動算出

---

### 変更・中途解約時の再計算

`f_HEN_SCH`（返済スケジュール変更）画面から変更が登録されると、変更日以降の償却・債務スケジュールが再計算される。変更前後の差額は `D_HENL`（変更履歴）に記録される。

## H2-7: 仕訳出力の仕組み

リースM4BSは3種類の仕訳を出力し、それぞれ標準フォームと顧客固有フォームの両方が存在する。

### 3種類の仕訳

| 種別 | 仕訳名 | 対応フォーム | 内容 |
|------|--------|------------|------|
| 計上仕訳（KEIJO） | 月次計上仕訳 | `f_仕訳出力標準_KJ` | リース費用・資産償却の会計仕訳 |
| 支払仕訳（TOUGETSU） | 支払管理仕訳 | `f_仕訳出力標準_SM` | リース料支払時の仕訳（支払管理経由） |
| 支払仕訳（TOUGETSU） | 支払処理仕訳 | `f_仕訳出力標準_SH` | リース料支払時の仕訳（支払処理経由） |
| 経費仕訳 | 経費計上 | 経費関連フォーム | 経費精算・配賦仕訳 |

---

### 共通構造（Form_Open パターン）

全仕訳出力フォームは以下の共通手順で初期化される。

```
Form_Open()
  1. 仕訳条件テーブル検証
     - 支払仕訳: tw_S_TOUGETSU_JOKEN
     - 計上仕訳: tw_S_KEIJO_JOKEN
  2. 期間チェック
     - KIKAN_FROM / KIKAN_TO の整合性確認
  3. 計算パラメータ検証
     - KTMG（計上目的）
     - TAISHO（対象区分）
     - MEISAI（明細区分）
  4. ワークテーブル初期化
     - tw_fc_支払仕訳 / tw_fc_計上仕訳 をクリア
```

設定は `f_仕訳出力標準_設定_MAIN/SM/SH/KJ` の各設定画面で管理する。

---

### 出力形式

| 形式 | 概要 | 対応顧客 |
|------|------|---------|
| Excel | Excelファイル出力（Interop使用） | MYCOM, SNKO, JOT, RISO, NKSOL, VTC, VALQUA 等 |
| CSV | Shift_JIS CSV出力 | TSYSCOM |
| 固定長テキスト | 固定桁数のテキストファイル | KITOKU（CMSW2WRK等） |
| 標準テキスト/Excel | 標準仕訳（APGDH/APGDD/APGDS形式） | 標準仕訳（汎用） |

### 仕訳パターンマスタ（M_SWPTN）

## H2-8: セキュリティモデル

リースM4BSは独自のユーザー認証・権限管理機構を持ち、Access JETセキュリティに依存しない独立したセキュリティレイヤーを実装している。

### ユーザー認証

ログイン情報は `typ_gLogin` 構造体としてメモリ上で保持される。

```vba
Type typ_gLogin
    UserCD      As String   ' ユーザーコード
    PWD         As String   ' パスワード（暗号化済み）
    KngnCd      As String   ' 権限グループコード
    boAdmin     As Boolean  ' 管理者フラグ
    boMaster    As Boolean  ' マスタ更新権限フラグ
    boAPPROVAL  As Boolean  ' 承認権限フラグ
    boLOG       As Boolean  ' ログ参照権限フラグ
    boFOutKNGN  As Boolean  ' ファイル出力権限フラグ
    boPriKNGN   As Boolean  ' 印刷権限フラグ
End Type
```

パスワードは `pc_Encrypt` 関数で暗号化して `SEC_USER` テーブルに保存される。

---

### パスワードポリシー

| 項目 | 内容 |
|------|------|
| ログイン試行制限 | 最大 5 回（`cn_gLOGIN_ATTEMPTS_SYS = 5`）超過でアカウントロック |
| パスワード有効期限 | T_SYSTEM で設定（日数指定） |
| 猶予期間 | 有効期限後も一定期間ログイン可能な猶予日数 |
| 最小文字数 | T_SYSTEM で設定 |
| 文字種チェック | 英字・数字・記号の混在要否を設定可能 |

---

### アクセス権限体系（ACCESS_KIND）

アクセス種別は3段階で管理される。

| ACCESS_KIND | 区分名 | 内容 |
|-------------|--------|------|
| 1 | 全データ更新 | 全契約管理単位・全部門のデータを更新可能 |
| 2 | 全データ参照 | 全データを参照のみ可能（更新不可） |
| 3 | 管理単位限定 | 自ユーザーに紐づく契約管理単位・部門のみアクセス可能 |

ACCESS_KIND=3（管理単位限定）の場合、以下のリストで制限範囲を管理する。

- `tgKKNRI_LIST`: 契約管理単位の許可リスト
- `tgBKNRI_LIST`: 部門管理単位の許可リスト

---

### 予約ユーザー

| ユーザーID | ユーザーコード | 用途 |
|-----------|--------------|------|
| 1 | system | システム管理者（全権限、顧客使用不可） |
| 0 | iltex | イルテックス保守専用アカウント |

---

### メニュー表示制御

権限グループに応じてメニューの表示/非表示を動的に制御する。

```
mMENUBAR_ENABLE_CTL()
  ├─ tcon_MenuMatrix から権限グループ別メニュー定義を取得
  ├─ ACCESS_KIND・各boフラグを評価
  └─ 非許可メニュー項目をグレーアウト/非表示
```

メニューは5階層構造（`tcon_MenuMatrix`）で定義されており、起動時に `p_StartUp_gMain` がメニューバーを動的生成する。

---

### 操作ログ

| ログ種別 | テーブル | 記録内容 |
|---------|---------|---------|
| 操作ログ | `L_SLOG` | 画面開閉・主要ボタン操作の履歴 |
| 更新ログ | `L_ULOG` | データ変更前後の値（フィールド単位） |
| バックアップログ | `L_BKLOG` | バックアップ実行履歴 |

ログは Tempフォルダ配下の専用MDB（`LM42log.mdb`）に格納され、起動時に初期化される。ログ参照は `boLOG=True` の権限を持つユーザーのみ可能。
