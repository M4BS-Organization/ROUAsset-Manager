## 2. ログイン画面が必要な理由

### 2-1. 要件定義との整合性

要件定義書（F-BZ-005）で「認証・権限管理」が**優先度 高**として明記されている。現状は未実装であり、ToBe で対応すべき項目。

### 2-2. 業務上の必要性

総務経理_業務分担.md より:
- 「総務部が入力、経理部が確認」のワークフローが存在
- 部門別の権限分離が運用上求められている
- システム管理者によるマスタ管理・ユーザー管理が将来計画に含まれる

### 2-3. セキュリティ上の必要性

- マルチユーザー対応のクライアントサーバー型（最大50ユーザー）
- PostgreSQL接続情報が共有されるため、アプリレベルでの認証が必要
- 監査証跡（誰がいつ操作したか）の記録に、ログインユーザーIDが必要

## 3. 認証方式の選択肢

### 方式A: アプリケーション独自認証（ID/パスワード）

```
アプリ起動 → FrmLogin（ログイン画面）→ tm_USER で認証 → FrmFlexMenu
```

| 項目 | 評価 |
|---|---|
| 要件定義との整合性 | ○ 完全一致（AuthorizationService + tm_USER） |
| 実装コスト | 中（ログイン画面 + 認証ロジック + パスワードハッシュ化） |
| ユーザー管理 | マスタ管理画面で管理可能 |
| 柔軟性 | ○ 権限レベルを自由に設計可能 |
| セキュリティ | △ パスワード管理の責任がアプリ側に |

**必要な実装:**
1. FrmLogin フォーム（ログイン画面UI）
2. tm_USER テーブルへのパスワードカラム追加（ハッシュ化）
3. AuthorizationService（認証＋権限チェック）
4. セッション管理（ログインユーザー情報の保持）
5. マスタ管理画面へのユーザー管理タブ追加

### 方式B: Windows認証（Active Directory連携）

```
アプリ起動 → OS認証情報を自動取得 → tm_USER でユーザー照合 → FrmFlexMenu
```

| 項目 | 評価 |
|---|---|
| 要件定義との整合性 | △ AuthorizationService は使えるが、ID/パスワード認証ではない |
| 実装コスト | 低（Windows認証は既に AuthenticationMode.Windows で設定済み） |
| ユーザー管理 | Active Directory 側で管理（IT部門の協力が必要） |
| 柔軟性 | △ AD環境に依存 |
| セキュリティ | ○ パスワード管理はOS側 |

**必要な実装:**
1. Windows ログインユーザー名の取得（`Environment.UserName`）
2. tm_USER テーブルとの照合（Windows ユーザー名 → アプリ権限のマッピング）
3. AuthorizationService（権限チェック）
4. 未登録ユーザーのアクセス拒否処理

### 方式C: ハイブリッド（Windows認証 + アプリ権限管理）

```
アプリ起動 → OS認証情報を自動取得 → tm_USER で権限確認
  ├─ 登録ユーザー → 権限に応じたメニュー表示
  └─ 未登録ユーザー → アクセス拒否 or ゲストモード
```

| 項目 | 評価 |
|---|---|
| 要件定義との整合性 | ○ 認証はWindows、権限管理はアプリ |
| 実装コスト | 低〜中 |
| ユーザー管理 | パスワード管理不要、権限のみアプリで管理 |
| 柔軟性 | ○ AD環境でもスタンドアロンでも対応可能 |
| セキュリティ | ○ |

## 4. 推奨案

### 推奨: 方式A（アプリケーション独自認証）

**理由:**
1. 要件定義書（F-BZ-005）が「ユーザーID/パスワード認証」を明記している
2. 基本設計書が `AuthorizationService` + `tm_USER` テーブルでの認証を前提としている
3. Active Directory 環境に依存しない（導入先の環境を問わない）
4. 権限レベル（総務部/経理部/管理者）を柔軟に設定可能

### 推奨する画面フロー

```
アプリケーション起動
  ↓
FrmLogin（ログイン画面）
  ├─ ユーザーID入力
  ├─ パスワード入力
  ├─ [ログイン] ボタン
  │    ↓
  │  AuthorizationService.Authenticate()
  │    ├─ 成功 → FrmFlexMenu（権限に応じたメニュー制御）
  │    └─ 失敗 → エラーメッセージ（3回失敗でロックも検討）
  └─ [終了] ボタン → アプリケーション終了
```

### 推奨する権限レベル

| 権限レベル | 対象 | アクセス可能画面 |
|---|---|---|
| 管理者 | IT部門・システム管理者 | 全画面 + ユーザー管理 |
| 経理担当 | 経理部 | 全業務画面（参照 + 登録・編集） |
| 総務担当 | 総務部 | 契約書入力 + 使用権資産（参照のみ） |
| 照会専用 | 監査・管理職 | 全画面（参照のみ） |
