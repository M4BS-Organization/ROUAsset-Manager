### 1.3 依存関係の方向

```
UI層 → Business層 → DataAccess層 → PostgreSQL
```

- 各層は下位の層にのみ依存し、逆方向の依存は禁止する
- 層間はインターフェースを介して疎結合に接続する
- 依存関係の構築は Composition Root (Program.vb) に集約する

### 1.4 MVPパターン詳細

#### 採用理由

| 検討パターン | 評価 | 採否 |
|---|---|---|
| MVP (Passive View) | Windows Formsに最適・テスト容易・既存パターンとの親和性高 | **採用** |
| MVVM | Windows Formsのデータバインディング不十分 | 不採用 |
| MVC | Windows Forms向けではない | 不採用 |

#### View-Presenter間の通信

```
View (Form) ──── IView Interface ───── Presenter
     │                                      │
     │  SetData() / ShowError()             │
     └──────────────────────────────────────┘
                                            │
                                            │
                                     Service → Repository → DB
```

- **View**: UI表示のみ担当、ビジネスロジックを持たない
- **Presenter**: ビジネスロジックとUI更新の制御
- **Model**: DTO（Data Transfer Object）によるデータ受け渡し

## 5. セキュリティ設計

### 5.1 SQLインジェクション対策

全クエリでパラメータ化クエリを使用する。

```vb
' ○ 正しい実装
Dim sql = "SELECT * FROM users WHERE name = @name"
Dim params = New Dictionary(Of String, Object) From {{"@name", inputValue}}
crud.ExecuteSelect(sql, params)

' × 禁止（文字列連結）
Dim sql = "SELECT * FROM users WHERE name = '" & inputValue & "'"
```

### 5.2 動的SQL構築のセキュリティ

FlexDataService における動的WHERE/ORDER BY句生成のセキュリティ対策:

1. **ホワイトリスト検証**: FlexDefinitionで定義されたSearchEnabled/SortEnabledカラムのみ使用可能
2. **演算子検証**: 事前定義リスト（=, LIKE, >, <, >=, <=, <>）のみ許可
3. **パラメータ化**: 検索値は全てNpgsqlパラメータとして渡す
4. **カラム名エスケープ**: PostgreSQL識別子を二重引用符で囲む

### 5.3 認証・認可

| 機能 | 実装方法 |
|---|---|
| ユーザー認証 | tm_USERテーブルによるID/パスワード認証 |
| メニュー権限 | ts_MENUのITEM_ENABLED + AuthorizationService |
| 機能アクセス制御 | AuthorizationServiceによるユーザー権限チェック |

## 6. エラーハンドリング設計

### 6.1 エラー処理フロー

```
PostgreSQL 例外
    │ DataAccessErrorMapper
    │ PostgresException → MapFromPostgres() → QueryExecutionException
    │ NpgsqlException → MapFromNpgsql() → DatabaseConnectionException
    │ Exception → MapFromGeneral() → DataAccessException
    │
DataErrorCollection に蓄積
    │
ErrorOccurred イベント発火 (将来のログ機能用)
    │
Presenter でキャッチ → View にエラー表示
```

### 6.2 エラーコードによる分岐パターン

```vb
Try
    crud.ExecuteInsert(sql, parameters)
Catch ex As QueryExecutionException
    Select Case ex.ErrorCode
        Case DataAccessErrorCode.UniqueViolation
            ' → 「このデータは既に登録されています」
        Case DataAccessErrorCode.ForeignKeyViolation
            ' → 「関連データが存在しません」
        Case DataAccessErrorCode.NotNullViolation
            ' → 「必須項目が入力されていません（カラム: {ex.ColumnName}）」
        Case Else
            ' → 「予期しないエラーが発生しました」
    End Select
End Try
```

### 6.3 DAO互換エラー対応表

| DAO | VB.NET (DataAccessErrors) |
|---|---|
| Err.Number | DataAccessErrorCode |
| Err.Description | DataError.Message |
| Err.Source | DataError.Source |
| DAO.Errors コレクション | DataErrorCollection |
| DAO.Error | DataError |

### 8.1 Binary If 演算子の制約

```vb
' ○ Nullable型 → Binary If 使用可
parameters.Add("@nullable_int", If(entity.NullableInt, DBNull.Value))

' ○ 参照型（String）→ Binary If 使用可
parameters.Add("@name", If(entity.Name, DBNull.Value))

' × 非Nullable値型（Integer, Boolean）→ Binary If 使用不可
' 直接代入すること
parameters.Add("@bpnid", entity.Bpnid)
parameters.Add("@calc_k", entity.CalcK)
```
