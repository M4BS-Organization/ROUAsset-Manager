# 要件定義: 固定長出力フォーマット (#30)

## 1. 目的
KITOKU顧客向け固定長テキスト出力の4フォーマットをVB.NETで実装し、Access版と完全一致する出力を保証する。

## 2. 対象フォーマット

### 2.1 CMSW2WRK (伝票ワーク)
- フィールド数: 52
- レコード長: 571バイト
- 出力ファイル: CMSW2WRK.tmp

### 2.2 APGDHWRK (金額概要ワーク)
- フィールド数: 72
- レコード長: 計算結果に基づく
- 出力ファイル: APGDHWRK.tmp

### 2.3 APGDDWRK (金額詳細ワーク)
- フィールド数: 31
- レコード長: 計算結果に基づく
- 出力ファイル: APGDDWRK.tmp

### 2.4 APGDSWRK (支払ワーク)
- フィールド数: 62
- レコード長: 計算結果に基づく
- 出力ファイル: APGDSWRK.tmp

## 3. 機能要件

### FR-1: フォーマット定義
- 各フォーマットのフィールド名・バイト幅・数値フォーマットを定義するクラスを提供
- Access版VBAと完全に一致するフィールド順序・バイト幅

### FR-2: ファイルライター
- DataTableから固定長テキストファイルを出力
- Shift-JIS (cp932) エンコーディング
- 1レコード1行、改行コード付き
- 文字列: 右パディング（半角スペース）、超過時は切り捨て
- 数値: ゼロパディングフォーマット適用後にパディング

### FR-3: 数値フォーマット
| パターン | バイト幅 | 例 |
|---------|---------|---|
| `000000000000000000.000` | 22 | 12345.678 → "000000000000012345.678" |
| `00000.000000000000` | 18 | 1.5 → "00001.500000000000" |
| `00000` | 5 | 3 → "00003" |
| `0.0` | 22 | 0 → "0.0" + 19スペース |

### FR-4: Null/DBNull処理
- 文字列Null → 全バイトスペースパディング
- 数値Null → 0として扱い、フォーマット適用

## 4. 非機能要件

### NFR-1: Access版との互換性
- 同一入力データに対してバイト単位で完全一致する出力を生成すること

### NFR-2: テスト
- DB非依存のブラックボックステスト（DataTableに手動データセット）
- 各フォーマットの完全レコード一致検証
- パディング・切り捨て・数値フォーマットの個別検証
