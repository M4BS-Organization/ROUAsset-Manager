# 要件定義書: bugfix-remaining-todos

## 1. 機能概要

Issue #17「バグ修正 + 残TODO消化」は、Phase 1-2 で積み残されたバグと未実装 TODO を Phase 3（2026-03-19 の 1 日）に集中して解消するタスクである。
完了条件は「致命的バグが 0 件」であり、システムの基本業務フロー（ログイン → 契約参照 → 仕訳出力）が障害なく実行できる状態にすることを目的とする。

---

## 2. ユーザーストーリー

### US-001: 致命的バグの特定と修正

- **As a** 開発担当者
- **I want** 起動・ログイン・仕訳出力フローを妨げるバグを修正する
- **So that** ユーザーが基本業務をエラーなく完了できる

#### 受け入れ基準
- [ ] アプリケーションが正常に起動し、ログイン画面が表示される
- [ ] 有効な認証情報でログインすると Form_MAIN が表示される
- [ ] 月次仕訳計上フレックス（Form_f_flx_KEIJO）が集計条件を実行できる
- [ ] 仕訳出力標準（KJ / SM / SH）が Excel ファイルを正常に出力する
- [ ] 上記フロー中に未処理の例外（System.Exception のダイアログ）が発生しない

### US-002: Form_ContractEntry の契約修正機能（Update 処理）実装

- **As a** 契約担当者
- **I want** 既存契約レコードを修正して保存できる
- **So that** 契約情報の変更作業がデータ損失なく完了する

#### 受け入れ基準
- [ ] 修正モード（isNewMode = False）で保存ボタンを押すと、d_kykh テーブルが UPDATE される（INSERT ではない）
- [ ] UPDATE 成功時に「修正モードですが、Update 処理が未実装…」という警告ダイアログが表示されない
- [ ] 新規モード（isNewMode = True）の INSERT 動作に影響を与えない
- [ ] UPDATE 失敗時は適切なエラーメッセージが表示され、データが破損しない

### US-003: 未実装メニュー項目の無効化または代替表示

- **As a** エンドユーザー
- **I want** 未実装の機能にアクセスした際に明示的なフィードバックを得る
- **So that** 操作中に意図しない挙動が発生しない

#### 受け入れ基準
- [ ] Form_MAIN のシステムタブにある未実装メニュー（DB保存・復元・EXCEL取込・キャッシュクリア・DB最適化・DB作成・DB削除・ロック解除・DBマイグレーション・物件管理単位メンテナンス・データエクスポート・環境設定・統制オプション・パスワード変更・各種ログ管理・仕訳固定値設定）は現行の「未実装です」MessageBox を維持するか、メニュー項目を Enabled = False に設定する
- [ ] 致命的バグに直接関係しないメニューの「未実装」状態はスコープ外として容認される（仮定事項参照）

### US-004: NzInt の戻り値型安全性修正

- **As a** 開発担当者
- **I want** Utils.NzInt が Option Strict On 下でコンパイルエラーなく使用できる
- **So that** 型安全性が確保され、ランタイム型エラーが発生しない

#### 受け入れ基準
- [ ] `NzInt` の戻り値型が `Object` から `Integer` に変更されているか、呼び出し側で明示的キャストが行われている
- [ ] 変更後に Form_ContractEntry.vb など NzInt 呼び出し箇所でコンパイルエラーが発生しない
- [ ] `NzDec` および `NzDate` の戻り値型が適切な厳密型（`Decimal`, `DateTime`）を返している（現状 NzDec は Decimal を返すため変更不要）

### US-005: 回帰テストによる修正確認

- **As a** 開発担当者
- **I want** 修正後に既存の自動テストを全件パスさせる
- **So that** Phase 1-2 で実装した機能が壊れていないことを確認する

#### 受け入れ基準
- [ ] 既存の E2E ブラックボックステスト（test_e2e_blackbox.exe）が PASS する
- [ ] f_CHUKI_JOKEN + f_IDOLST_JOKEN ブラックボックステスト（test_chuki_idolst_joken_blackbox.exe）が PASS する
- [ ] f_KEIJO_JOKEN ブラックボックステスト（test_keijo_joken_blackbox.exe）が PASS する
- [ ] スケジュールブラックボックステスト（test_schedule_blackbox.exe）が PASS する
- [ ] 型安全性ブラックボックステスト（test_type_safety_blackbox.exe）が PASS する

## 3. 機能要件

### FR-001: 致命的バグ分類と対応義務
- 説明: 以下の条件を満たす不具合は「致命的バグ」として Phase 3 内に修正が必須。
  (1) アプリケーションの起動失敗
  (2) ログインフロー（認証 → セッション確立 → メインメニュー表示）の完全停止
  (3) 月次仕訳計上および仕訳出力フローで未処理例外が発生しデータが出力されない
  (4) データ破損を引き起こす（Insert/Update の誤動作など）
- 優先度: 必須

### FR-002: 重大バグの対応
- 説明: 業務継続は可能だが、特定操作で頻繁にエラーが発生する不具合は「重大バグ」とし、時間が許す範囲で修正する。修正できない場合は Issue にコメントを残す。
- 優先度: 推奨

### FR-003: 軽微バグ・未実装 TODO の対応方針
- 説明: 表示乱れ、未実装メニューの MessageBox 維持、コメント整理など業務に直接影響しない事項は「軽微バグ」とし、今 Phase では対応不要とする。
- 優先度: 任意

### FR-004: Form_ContractEntry の Update 処理実装
- 説明: `isNewMode = False` の保存時に CrudHelper.Update を使用して d_kykh を更新する。CrudHelper には既に `Update(tableName, columnValues, whereClause, Optional whereParameters)` が実装済みであるため、フォーム側のコードを修正するのみでよい。
- 優先度: 必須

### FR-005: NzInt 関数の戻り値型修正
- 説明: `Utils.NzInt` の戻り値型が `Object` になっており、Option Strict On 環境で暗黙ダウンキャストが発生している。戻り値型を `Integer` に変更し、呼び出し側の型一致を保証する。
- 優先度: 必須（Option Strict On が有効な場合はコンパイルエラー）

### FR-006: 未実装メニューの状態維持または明示的無効化
- 説明: Form_MAIN のシステムタブ配下の未実装 17 件のメニュー項目は、現行の MessageBox 表示を維持する。別 Phase での実装が予定されているため、メニュー項目自体は削除しない。
- 優先度: 推奨（現状が致命的バグでないため優先度は低い）

## 4. 非機能要件

### NFR-001: パフォーマンス
- 仕訳出力（Form_f_仕訳出力標準_KJ / SM / SH）の Excel 出力処理は、標準的な契約データ 100 件に対して 30 秒以内に完了すること（Access 版と同等水準）。

### NFR-002: セキュリティ
- `DbConnectionManager.GetConnectionString()` のデフォルト接続文字列（フォールバック値 `Host=localhost;...`）はテスト環境向け記述であり、本番環境では App.config または環境変数 `LEASE_M4BS_CONNECTION_STRING` で上書きする。このフォールバック自体は現 Phase では変更しない（仮定事項参照）。
- パスワードが接続文字列にハードコードされている箇所について、Phase 3 の 1 日の範囲内では修正しないが、バグとして Issue にコメントを残す。

### NFR-003: ユーザビリティ
- 修正後の画面操作において、修正モードでの保存時に誤った警告ダイアログが表示されないこと。
- エラー発生時は「エラーの種類とメッセージ」を含む MessageBox が表示され、ユーザーが次の行動を判断できること。

### NFR-004: 接続管理
- Npgsql 接続は全て `Using` ブロック内で管理されていること。フォーム修正時に新たな接続リークを導入しないこと。
- CrudHelper のトランザクション管理（BeginTransaction / CommitTransaction / RollbackTransaction）が利用可能であり、Update 処理でトランザクションが必要な場合は活用する。

### NFR-005: 型安全性
- 修正対象ファイルは Option Strict On / Option Explicit On の設定下でコンパイルエラーが 0 件であること。
- 既存ファイルで Option Strict On が設定されていない場合、修正対象ファイルのみに `Option Strict On` を追加し、他ファイルへの波及は Phase 3 の範囲外とする。

## 5. 前提条件・制約

- Phase 3 は 2026-03-19 の 1 日（実質数時間）で作業完了する必要がある。
- VB.NET (.NET Framework 4.7.2), WinForms, Npgsql を使用しており、フレームワーク変更はスコープ外。
- テスト環境の PostgreSQL インスタンスが稼働していることを前提とする。
- CrudHelper.Update メソッドは既に実装済みであり、Form 側の修正のみで対応可能（調査済み: `LeaseM4BS.DataAccess\CrudHelper.vb:283`）。
- 未実装メニュー 17 件は Phase 3 での実装対象外であり、現状の MessageBox 維持が許容される（完了条件に含まれない）。

- 接続文字列のハードコード解消（本番環境では App.config / 環境変数で対応）
- Phase 1-2 未完了の機能の新規実装（Issue #11〜#13 相当）
- 全ファイルへの Option Strict On 一括適用（修正対象ファイルのみ対応）
- データベーススキーマ変更
- UI レイアウトの改善

## 7. 用語定義

| 用語 | 定義 |
|---|---|
| 致命的バグ | アプリケーション起動・ログイン・主要業務フロー（仕訳出力）を完全に停止させる不具合 |
| 重大バグ | 業務継続は可能だが特定操作で頻繁にエラーが発生する不具合 |
| 軽微バグ | 表示崩れや操作の不便さなど、業務継続に影響しない不具合 |
| 未実装 TODO | コード内で MessageBox「未実装です」または警告として残されている処理 |
| CrudHelper | PostgreSQL に対する汎用 CRUD 操作ヘルパークラス（`LeaseM4BS.DataAccess`） |
| NzInt / NzDec / NzDate | Access VBA の `Nz()` に相当する NULL 安全変換ユーティリティ関数 |
| Option Strict On | VB.NET コンパイラオプション。暗黙の型変換を禁止し型安全性を強制する |
| isNewMode | Form_ContractEntry で新規登録(True)・修正(False)を制御するフラグ |

## 8. 仮定事項

1. **致命的バグ 0 件の検証方法（仮定）**: E2E ブラックボックステスト（test_e2e_blackbox.exe）がログイン→契約参照→仕訳出力の全フローを網羅しているため、同テストの全 PASS をもって「致命的バグ 0 件」と判断する。E2E テストがカバーしていない画面は目視確認で補完する。

2. **未実装メニューのスコープ（仮定）**: Form_MAIN の未実装 17 件はシステムタブ配下であり、完了条件「致命的バグ 0 件」には含まれないと判断した。別 Issue / 別フェーズで対応予定。実装前に担当者（小谷）と確認が必要。

3. **NzInt の修正影響範囲（仮定）**: `NzInt` の戻り値型変更は Utils.vb と呼び出し元（Form_ContractEntry.vb 等）の変更を伴う可能性がある。呼び出し箇所全件をコンパイラエラーで洗い出し、修正する。

4. **接続文字列フォールバック（仮定）**: `DbConnectionManager.vb:76` のデフォルト接続文字列はテスト環境用として現状維持とする。本番デプロイ前に App.config で上書きすることを前提とする。実装前にプロジェクトオーナーへの確認を推奨。

5. **回帰テスト実行環境（仮定）**: ブラックボックステスト (.exe) は Windows 環境で直接実行可能であり、CI（GitHub Actions: ci.yml）でも MSBuild ビルド + テスト実行が構成されていると仮定する。テスト実行前に PostgreSQL 接続の確認が必要。
