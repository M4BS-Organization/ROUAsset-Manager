# 使用権資産管理画面の移植 — 作業調査・Issue分解資料

> **作成日:** 2026-03-23
> **対象プロジェクト:** LeaseM4BS（リース資産管理システム）
> **移植元:** ROUAsset-Manager-old（使用権資産管理アプリ）

## 1. 目的

旧リポジトリ（ROUAsset-Manager-old）で作成途中の使用権資産管理画面群を、新リポジトリ（ROUAsset-Manager）に移植し、既存の契約フレックス画面（`Form_f_flx_CONTRACT`）から使用権資産管理画面へ遷移できるようにする。

## 3. 移植方針の検討

### 3.1 2つのアプローチ

| 方針 | 内容 | メリット | デメリット |
|---|---|---|---|
| **A: そのまま移植** | 旧リポジトリのファイルを新リポジトリにコピーし、最小限の改修で動作させる | 早い、動作実績あり | 新リポジトリのパターンと不統一 |
| **B: 新パターンに書き換え** | 旧画面を新リポジトリの設計パターン（FormHelper, CrudHelper, LoginSession等）に合わせて再実装 | 統一性、保守性 | 工数大 |

### 3.2 推奨: ハイブリッドアプローチ（A寄り）

1. **UI層（Form/UserControl）**: そのままコピーし、最小限の名前空間・参照調整で動作させる
2. **データアクセス層**: 新リポジトリの `CrudHelper` を使うようにアダプタを作成
3. **認証・権限**: 新リポジトリの `LoginSession` に合わせて権限チェックを差し替え
4. **画面遷移**: 新リポジトリの `Form_f_flx_CONTRACT` → 旧 `FrmLeaseContractMain` への導線を追加

## 4. 主要GAP分析

### 4.1 画面遷移パターンの違い

**新リポジトリ（Form_f_flx_CONTRACT → Form_f_KYKH）**:
```vb
' 一覧からダブルクリックで契約詳細を開く
Private Sub dgv_LIST_CellDoubleClick(...)
    Dim frm As New Form_f_KYKH()
    frm.SetParams(kykhId)
    frm.ShowDialog()
End Sub
```

**旧リポジトリ（FrmFlexContract → FrmLeaseContractMain）**:
```vb
Private Sub OpenContractMain(contractNo As String, mode As ContractOpenMode)
    Dim frm As New FrmLeaseContractMain()
    frm.InitContractNo = GetNextContractNo()  ' NewEntry時
    frm.Tag = contractNo                       ' Edit/Inquiry時
    AddHandler frm.ContractRegistered, AddressOf OnContractRegistered
    frm.Show()  ' 非モーダル
End Sub
```

**GAP**: ShowDialog (モーダル) vs Show (非モーダル)。新リポジトリのパターンに合わせて `ShowDialog()` に変更推奨。

### 4.2 データモデルの違い

**新リポジトリ**: `d_kykh`（契約ヘッダ）+ `d_kykm`（物件）+ 多数の関連テーブル
```sql
-- 55テーブル構成。主要テーブル:
d_kykh (リース契約ヘッダ)
d_kykm (物件/リース期間)
d_bukn_haif (配分情報)
d_bukn_gson (月額)
```

**旧リポジトリ**: `ctb_lease_integrated`（統合1テーブル）+ `ctb_dept_allocation`
```sql
-- 2テーブル構成（フラットな統合テーブル）
ctb_lease_integrated (契約×資産×基本情報を1行に統合)
ctb_dept_allocation (配賦部門)
```

**GAP**: テーブル設計が根本的に異なる。移植時に以下のいずれかが必要:
- (a) `ctb_*` テーブルを新DBにも追加（独立運用）
- (b) `d_kykh` / `d_kykm` 等の既存テーブルに対応するようCtbRepositoryを書き換え
- **(c) 推奨: まずは(a)で動作させ、段階的に(b)へ移行**

### 4.3 ヘルパー・ユーティリティの違い

| 機能 | 新リポジトリ | 旧リポジトリ | 移植時の対応 |
|---|---|---|---|
| ComboBoxバインド | `FormHelper.Bind()` 拡張メソッド | 直接 `MasterDataLoader` で手動セット | そのまま移植可 |
| DataGridView操作 | `FormHelper.GetSelectedRow()` | 直接 `SelectedRows(0)` | そのまま移植可 |
| DB接続 | `DbConnectionManager`（共通） | `DbConnectionManager`（同名・同設計） | **互換性あり** |
| CRUD | `CrudHelper`（共通） | `CtbRepository`（専用） | アダプタ追加 |
| セッション | `LoginSession` Module | `AuthorizationService` クラス | 差し替え必要 |
| セキュリティ | `SecurityChecker` | `AuthorizationService.HasAccess()` | 差し替え必要 |
| マスタ読込 | 各フォームが直接SQL | `MasterDataLoader` | `MasterDataLoader` も一緒に移植 |

### 4.4 名前空間・参照の違い

| 項目 | 新リポジトリ | 旧リポジトリ |
|---|---|---|
| UI名前空間 | `LeaseM4BS.TestWinForms` | `LeaseM4BS.TestWinForms` |
| DA名前空間 | `LeaseM4BS.DataAccess` | `LeaseM4BS.DataAccess` |
| NuGet | Npgsql 6.0.11 | Npgsql 6.0.11 |
| Framework | .NET Framework 4.7.2 | .NET Framework 4.7.2 |

**好材料**: 名前空間・フレームワーク・NuGetが完全一致。コピーだけで参照解決できる可能性が高い。

```
□ ソリューションビルドが通ることを確認
```

### 8.2 認証・権限の差し替えポイント

旧リポジトリの `AuthorizationService` は新リポジトリに存在しないため、以下の箇所を `LoginSession` に差し替え:

```vb
' 旧: AuthorizationService.Current.HasAccess("btnContract")
' 新: LoginSession.CanMasterUpdate / LoginSession.AccessKind 等で判定

' 旧: AuthorizationService.Current.CurrentUser.DisplayName
' 新: LoginSession.LoggedInUserNm
```

### 8.3 画面遷移の統一

新リポジトリのパターンに合わせて `ShowDialog()` を使用:

```vb
' Form_f_flx_CONTRACT.vb に追加するコード例
Private Sub btnROUAsset_Click(sender As Object, e As EventArgs)
    Dim selectedRow = dgv_LIST.GetSelectedRow()
    If selectedRow Is Nothing Then Return

    Dim kykhId As Integer = CInt(selectedRow.Cells("kykh_id").Value)
    Dim frm As New FrmLeaseContractMain()
    frm.KykhId = kykhId  ' 新しいプロパティ
    frm.ShowDialog(Me)
End Sub
```

### 8.4 DBテーブルの共存

`ctb_*` テーブルと `d_kykh` / `d_kykm` テーブルは独立して共存可能:
- 初期段階: `ctb_*` テーブルを使用（旧ロジックそのまま）
- 将来: `d_kykh` / `d_kykm` への統合時に CtbRepository をリファクタリング
