## 2. Step 1: tm_USER テーブル拡張

### 2-1. 追加カラム定義

| カラム名 | データ型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| `login_id` | VARCHAR(50) | NOT NULL | — | ログイン用ID（UNIQUE、ログイン画面で入力） |
| `password_hash` | VARCHAR(256) | NOT NULL | — | BCrypt ハッシュ化パスワード |
| `role` | VARCHAR(20) | NOT NULL | `'viewer'` | 権限ロール（後述） |
| `is_active` | BOOLEAN | NOT NULL | `TRUE` | アカウント有効フラグ |
| `last_login_at` | TIMESTAMP | NULL | — | 最終ログイン日時 |
| `failed_login_count` | INTEGER | NOT NULL | `0` | ログイン連続失敗回数 |
| `locked_until` | TIMESTAMP | NULL | — | アカウントロック解除日時（Phase 2 で利用） |

> **login_id と user_id の使い分け:** tm_USER の既存 PK `user_id` は SERIAL（自動採番）であり、
> 内部的な参照キーとして使用する。ユーザーがログイン画面で入力するIDは `login_id`（文字列、UNIQUE）。

### 2-2. DDL

```sql
-- tm_USER テーブル拡張
-- ※ 既存行が存在する可能性があるため、password_hash は一時的に DEFAULT 付きで追加後、制約変更する
ALTER TABLE tm_USER ADD COLUMN login_id VARCHAR(50);
ALTER TABLE tm_USER ADD COLUMN password_hash VARCHAR(256);
ALTER TABLE tm_USER ADD COLUMN role VARCHAR(20) NOT NULL DEFAULT 'viewer';
ALTER TABLE tm_USER ADD COLUMN is_active BOOLEAN NOT NULL DEFAULT TRUE;
ALTER TABLE tm_USER ADD COLUMN last_login_at TIMESTAMP;
ALTER TABLE tm_USER ADD COLUMN failed_login_count INTEGER NOT NULL DEFAULT 0;
ALTER TABLE tm_USER ADD COLUMN locked_until TIMESTAMP;

-- 既存行のマイグレーション（既存ユーザーがいる場合）
UPDATE tm_USER SET login_id = user_name WHERE login_id IS NULL;
UPDATE tm_USER SET password_hash = '<初期BCryptハッシュ>' WHERE password_hash IS NULL;

-- NOT NULL 制約を後から追加（既存行の更新後に実行）
ALTER TABLE tm_USER ALTER COLUMN login_id SET NOT NULL;
ALTER TABLE tm_USER ALTER COLUMN password_hash SET NOT NULL;

-- ユニーク制約・チェック制約
ALTER TABLE tm_USER ADD CONSTRAINT uq_user_login_id UNIQUE (login_id);
ALTER TABLE tm_USER ADD CONSTRAINT chk_user_role
  CHECK (role IN ('admin', 'accounting', 'general_affairs', 'viewer'));

-- 初期管理者ユーザー投入（パスワードは初回ログイン時に変更を促す）
-- ※ ハッシュ値はアプリ起動後にツールで生成
INSERT INTO tm_USER (user_name, user_kana, login_id, password_hash, role, is_active)
VALUES ('管理者', '管理者', 'admin', '<BCryptハッシュ>', 'admin', TRUE);
```

### 2-3. ロール定義

| ロール値 | 対象部門 | 説明 |
|---|---|---|
| `admin` | IT部門・システム管理者 | 全画面 + ユーザー管理 |
| `accounting` | 経理部 | 全業務画面（参照 + 登録・編集） |
| `general_affairs` | 総務部 | 契約書入力 + 使用権資産（参照のみ） |
| `viewer` | 監査・管理職 | 全画面（参照のみ） |

## 3. Step 2: AuthorizationService

### 3-1. クラス設計

**配置先:** `LeaseM4BS.DataAccess/AuthorizationService.vb`

基本設計書 §3.4 で Business 層クラスとして定義済み。既存の `CrudHelper` パターンに従い、パラメータ化クエリで実装する。

### 3-2. パブリック API

```vb
Public Class AuthorizationService

    ''' <summary>ログインユーザー情報を保持するシングルトン</summary>
    Public Shared ReadOnly Property Current As AuthorizationService

    ''' <summary>現在ログイン中のユーザー情報</summary>
    Public ReadOnly Property CurrentUser As UserInfo  ' Nothing = 未ログイン

    ''' <summary>認証を実行する</summary>
    ''' <param name="loginId">ログインID（login_id カラム）</param>
    ''' <param name="password">平文パスワード</param>
    ''' <returns>認証結果（成功/失敗理由）</returns>
    Public Function Authenticate(loginId As String, password As String) As AuthResult

    ''' <summary>指定メニューへのアクセス権があるか判定</summary>
    Public Function HasAccess(menuId As String) As Boolean

    ''' <summary>ログアウト（セッションクリア）</summary>
    Public Sub Logout()

End Class
```

### 3-3. UserInfo クラス

```vb
Public Class UserInfo
    Public Property UserId As Integer       ' tm_USER.user_id (PK)
    Public Property LoginId As String       ' tm_USER.login_id（ログインに使用）
    Public Property UserName As String      ' tm_USER.user_name
    Public Property Role As String          ' "admin", "accounting", "general_affairs", "viewer"
    Public Property IsActive As Boolean
End Class
```

### 3-4. AuthResult 列挙型

```vb
Public Enum AuthResult
    Success = 0
    UserNotFound = 1
    InvalidPassword = 2
    AccountDisabled = 3
    AccountLocked = 4       ' Phase 2
End Enum
```

### 3-5. 認証フロー（疑似コード）

```vb
Public Function Authenticate(loginId As String, password As String) As AuthResult
    ' 1. tm_USER から login_id で該当ユーザーを検索（CrudHelper.GetDataTable）
    '    SQL: SELECT * FROM tm_USER WHERE login_id = @login_id
    ' 2. ユーザーが見つからない → UserNotFound
    ' 3. is_active = FALSE → AccountDisabled
    ' 4. BCrypt.Verify(password, password_hash) → 不一致なら InvalidPassword
    '    → failed_login_count をインクリメント（UPDATE）
    ' 5. 一致 → failed_login_count = 0 にリセット、last_login_at = Now を更新
    ' 6. CurrentUser に UserInfo をセット
    ' 7. Success を返す
End Function
```

### 3-6. パスワードハッシュ化

- **ライブラリ:** `BCrypt.Net-Next`（NuGet パッケージ）
- **ハッシュ生成:** `BCrypt.Net.BCrypt.HashPassword(plainPassword)`
- **検証:** `BCrypt.Net.BCrypt.Verify(plainPassword, storedHash)`
- **WorkFactor:** デフォルト（12）を使用

### 3-7. メニューアクセス制御マッピング

```vb
''' Phase 1 ではハードコードで実装する。
''' Phase 2 で ts_MENU テーブルの ITEM_ENABLED と連携させる
''' （基本設計書 §3.4「ts_MENUのITEM_ENABLED + AuthorizationService」に対応）。
Private Shared ReadOnly _menuPermissions As New Dictionary(Of String, String()) From {
    {"btnContract",          New String() {"admin", "accounting", "general_affairs", "viewer"}},
    {"btnROUAsset",          New String() {"admin", "accounting", "general_affairs", "viewer"}},
    {"btnMonthlyPayments",   New String() {"admin", "accounting", "viewer"}},
    {"btnMonthlyAccounting", New String() {"admin", "accounting", "viewer"}},
    {"btnPeriodBalance",     New String() {"admin", "accounting", "viewer"}},
    {"btnTaxAdjustment",     New String() {"admin", "accounting", "viewer"}},
    {"btnMaster",            New String() {"admin"}}
}
' 注: general_affairs は月次系メニューにアクセス不可（Enabled=False）
' 注: viewer は全業務メニューにアクセス可能だが、
'     各画面内での編集・保存ボタンは無効化（Phase 2 で詳細制御）
```

> **Phase 2 での ts_MENU 連携方針:** ハードコードの `_menuPermissions` を廃止し、
> ts_MENU テーブルの `ITEM_ENABLED` フラグとロール別の権限マッピングテーブルで動的に制御する。

### 4-4. イベント処理

```vb
' ログインボタンクリック
Private Sub btnLogin_Click(sender As Object, e As EventArgs) Handles btnLogin.Click
    ' 1. 入力バリデーション（空チェック）
    ' 2. AuthorizationService.Current.Authenticate(txtLoginId.Text, txtPassword.Text)
    ' 3. 結果に応じた処理:
    '    - Success → ShowMainMenu()
    '    - UserNotFound / InvalidPassword → lblError に "ログインIDまたはパスワードが正しくありません" 表示
    '    - AccountDisabled → lblError に "このアカウントは無効化されています" 表示
    ' 4. エラー時はパスワード欄をクリアし、txtLoginId にフォーカスを移動
End Sub

' 終了ボタンクリック
Private Sub btnExit_Click(sender As Object, e As EventArgs) Handles btnExit.Click
    Application.Exit()
End Sub
```

### 4-5. セキュリティ考慮事項

- エラーメッセージは「ログインIDまたはパスワードが正しくありません」で統一（ユーザー存在の推測を防止）
- パスワードは `PasswordChar="*"` + `UseSystemPasswordChar=True` で隠す
- 認証失敗時にパスワード欄をクリア

### 起動フロー（ToBe）

```
アプリケーション起動
  ↓

  ↓

  ↓ [ログイン成功]
```

### アプリケーション終了の制御

FrmLogin が MainForm のため、FrmLogin が閉じるとアプリ終了となる（`ShutDownStyle = AfterMainFormCloses`）。
FrmFlexMenu クローズ時に FrmLogin も閉じることで、正しいシャットダウンを実現する。

```vb
' FrmLogin 側
Private Sub ShowMainMenu()
    Me.Hide()
    Dim mainMenu As New FrmFlexMenu(AuthorizationService.Current.CurrentUser)
    AddHandler mainMenu.FormClosed, Sub(s, ev) Me.Close()
    mainMenu.Show()
End Sub
```

## 6. Step 5: FrmFlexMenu 権限制御

### 6-1. コンストラクタ変更

> **注意:** VB.NET の Windows Forms Designer はパラメータなしコンストラクタを必要とする。
> そのため、パラメータなしコンストラクタは残し、オーバーロードを追加する。

```vb
Private _currentUser As UserInfo

' Designer 用（パラメータなしコンストラクタを維持）
Public Sub New()
    InitializeComponent()
    SetupMenuButtons()
    SwitchContent(btnContract)
End Sub

' ★追加: ログイン後に使用するコンストラクタ
Public Sub New(currentUser As UserInfo)
    InitializeComponent()
    _currentUser = currentUser
    SetupMenuButtons()
    ApplyPermissions()           ' 権限に応じたメニュー制御
    SwitchContent(GetFirstAccessibleButton())  ' アクセス可能な最初のメニュー
End Sub
```

### 6-2. ApplyPermissions メソッド

```vb
Private Sub ApplyPermissions()
    ' AuthorizationService のメニューアクセス制御マッピングに基づき、
    ' 各ボタンの Enabled を設定
    For Each btn In {btnContract, btnROUAsset, btnMonthlyPayments,
                     btnMonthlyAccounting, btnPeriodBalance,
                     btnTaxAdjustment, btnMaster}
        btn.Enabled = AuthorizationService.Current.HasAccess(btn.Name)
    Next

    ' ステータスバーにログインユーザー情報を表示（任意）
    ' Me.Text = $"LeaseM4BS - {_currentUser.UserName} ({_currentUser.Role})"
End Sub
```

## 10. Phase 2（将来拡張）への申し送り

Phase 1 では以下を**意図的にスコープ外**とする:

| 項目 | Phase 2 での対応方針 |
|---|---|
| ユーザー管理画面 | btnMaster の管理画面にユーザー管理タブを追加 |
| パスワード変更機能 | FrmLogin にリンク追加 or 初回ログイン時に強制変更ダイアログ |
| アカウントロック | `failed_login_count >= 5` で `locked_until` を設定（カラムは Phase 1 で準備済み） |
| 画面内の編集制御 | viewer ロールの場合、各画面の保存・編集ボタンを Enabled=False |
| ts_MENU 連携 | ハードコードの権限マッピングを ts_MENU.ITEM_ENABLED と連携させる |
| 監査ログ | ログイン/ログアウト/主要操作を別テーブルに記録 |
| 自動ログアウト | 無操作タイマーでセッションタイムアウト |
