# ドキュメント調査資料 - 変更・異動・減損処理画面群

## 1. VB.NET WinForms DataGridView 複雑なデータ編集フォームのベストプラクティス

### EditMode設定
- `EditMode`プロパティで編集挙動を制御。`EditOnEnter`（セル選択時に即編集）、`EditOnKeystrokeOrF2`（キー入力/F2で編集開始）、`EditProgrammatically`（プログラムからのみ編集可）の3モード
- `ReadOnly`プロパティをセル/行/列/グリッド単位で設定可能

### データバインディング
- `BindingSource`コンポーネントを介してバインドするのが推奨パターン
- `CellEndEdit`イベントで編集完了を検知、`CellValueChanged`で値変更をハンドリング

### パフォーマンスベストプラクティス（大量データ時）
- セルスタイルは`DataGridView.DefaultCellStyle`レベルで設定（個別セルへの設定を避ける）
- 自動リサイズは`DisplayedCells`モードに限定（全行ではなく表示行のみ）
- 共有行（Shared Rows）を活用してメモリ効率を上げる
- 大量データ時は**Virtual Mode**を検討
- ContextMenuStripは`CellContextMenuStripNeeded`イベントで動的に提供

### 複雑な編集シナリオ
- カスタムセルタイプの作成で独自UIを組み込み可能
- `CellFormatting`イベントでスタイルを動的に適用

## 2. リース会計（IFRS16/ASC842）変更・異動・減損の会計処理フロー

### リース変更（Modification）の判定フロー

**Step 1: 独立した契約（Separate Contract）か判定**
以下の2条件を**両方**満たす場合 → 別個の契約として処理:
1. 変更により、元のリースに含まれていない追加の使用権（ROU）が付与される
2. リース料の増加が、追加使用権の独立販売価格に見合っている

**Step 2: 別個の契約でない場合 → リマジャメント（Remeasurement）**

### リマジャメント手順
1. **割引率の更新**: 変更日時点の割引率（増分借入利率）を使用
2. **リース負債の再測定**: 変更後のリース料・期間で再計算
3. **ROU資産の調整**: リース負債の変更額に応じてROU資産を調整
4. **損益認識**: ROU資産がゼロ以下になる場合、超過分を損益に計上

### 変更の種類と処理

| 変更タイプ | 処理 |
|-----------|------|
| スコープ拡大（追加使用権） | 別個の契約 or リマジャメント |
| スコープ縮小（部分解約） | 変更前のROU資産・負債を比例減少、差額を損益認識 |
| リース期間の延長/短縮 | 割引率更新 + リース負債再測定 + ROU資産調整 |
| リース料の変更 | リース負債再測定 + ROU資産調整 |
| 購入オプションの行使見込変更 | 割引率更新 + リース負債再測定 |

### IFRS16 vs ASC842 の主な違い
- **変動リース料（指数連動）**: IFRS16では指数/率の変更時にリース負債を再測定、ASC842ではトリガーイベント発生時のみ
- **リース分類**: IFRS16はファイナンスリースのみ（原則）、ASC842はファイナンス/オペレーティングの二分類を維持

### 減損（Impairment）
- ROU資産はASC360（有形固定資産）の減損指針に従う
- 減損は**資産のみ**に適用（負債は再測定しない）
- オペレーティングリースの場合、減損後は定額法ではなくなる

## 3. VB.NET での再計算ロジック実装パターン

### VB.NET組み込み金融関数（Microsoft.VisualBasic.Financial）
- `Financial.Pmt(rate, nper, pv, fv, due)` — 定額支払い額の算出
- `Financial.PPmt(rate, per, nper, pv, fv, due)` — 特定期間の元本部分
- `Financial.IPmt(rate, per, nper, pv, fv, due)` — 特定期間の利息部分

### 基本的な償却スケジュール計算ロジック
```
各期間について:
  利息 = 残存負債 × 期間利率
  元本返済 = 支払額 - 利息
  残存負債 = 残存負債 - 元本返済
```

### リマジャメント時の再計算パターン
1. 変更日時点の残存リース負債を取得
2. 新しい割引率（変更日時点）を適用
3. 残存期間の新しいリース料で新スケジュールを生成
4. ROU資産をリース負債の変更差額で調整
5. 新しい償却スケジュールを残存期間で再計算

### 注意点
- 月次計算の場合、年利率を12で除算: `monthlyRate = annualRate / 12`
- リマジャメント時は「変更前の残高」をスタートポイントとする

## 4. WinForms での変更履歴管理UIパターン

### 推奨アプローチ

**A. 監査証跡テーブル構造**
```
AuditTrail テーブル:
- ID (PK)
- TableName (変更対象テーブル)
- RecordID (変更対象レコードのキー)
- ActionType (INSERT/UPDATE/DELETE)
- FieldName (変更フィールド)
- OldValue (変更前の値)
- NewValue (変更後の値)
- ChangedBy (変更者)
- ChangedAt (変更日時)
```

**B. UIパターン**
1. **メイン画面 + 履歴タブ**: DataGridViewで現在データを表示し、タブコントロールで履歴一覧を表示
2. **変更前後の比較表示**: 左右分割または行内にOldValue/NewValueを色分け表示
3. **フィルタリング**: 日付範囲、変更者、変更種別でフィルタ

**C. 実装パターン**
- 自前実装の場合: DataRowのRowStateを監視し、BeforeUpdate/AfterUpdateイベントで監査レコードを生成
- 変更前のオブジェクトのディープコピーを保存し、変更後と比較してdiffを生成
