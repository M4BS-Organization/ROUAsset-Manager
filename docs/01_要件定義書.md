## 1. プロジェクト概要

### 1.1 背景

MS Access（.mdb）で構築されたスタンドアロン業務アプリケーション「シサンM7」は、長年にわたり固定資産管理業務に使用されてきた。しかし、同時アクセス制限、拡張性の限界、保守性の課題から、クライアントサーバー型アーキテクチャへの移行が必要となった。

また、新リース会計基準（IFRS16 / 日本基準）への対応として、既存のリースM4BSアプリケーションにリース判定機能を新規追加する必要がある。

### 1.2 プロジェクト目的

本プロジェクトの目的は以下の2点である。

1. **シサンM7移行**: MS Accessアプリケーション「シサンM7」を PostgreSQL + VB.NET（Windows Forms）のクライアントサーバー型アプリケーションへ移行し、マルチユーザー対応・保守性向上・拡張性確保を実現する。
2. **リースM4BS新リース会計基準対応**: IFRS および日本基準に基づくリース判定機能を実装し、使用権資産・リース負債の自動計算を可能とする。

### 1.3 対象システム

| 項目 | シサンM7 移行 | リースM4BS |
|---|---|---|
| 現行システム | MS Access (.mdb) スタンドアロン | リースM4BS（VB.NET Windows Forms） |
| 移行先 | PostgreSQL + VB.NET (Windows Forms) | 同一（機能追加） |
| 対象ユーザー | 固定資産管理担当者 | 経理・リース契約管理担当者 |

## 4. 非機能要件

### 4.1 性能要件

| 項目 | 要件 |
|---|---|
| 画面応答時間 | 一覧表示: 3秒以内、登録/更新: 2秒以内 |
| 同時接続数 | 最大50ユーザー（PostgreSQL接続プーリング） |
| データ量 | 421テーブル、将来的に数百万レコード対応 |

### 4.2 セキュリティ要件

| 項目 | 要件 |
|---|---|
| SQLインジェクション対策 | 全クエリでパラメータ化クエリを使用 |
| 動的SQL構築 | ホワイトリスト方式（許可テーブル名・カラム名のみ使用可能） |
| 演算子検証 | 使用可能な演算子を事前定義リストで制限 |
| カラム名エスケープ | PostgreSQL識別子を二重引用符でエスケープ |
| 認証 | ユーザーID/パスワード認証（AuthorizationService） |
| 権限制御 | メニュー項目の有効/無効制御、機能別アクセス制御 |

### 4.3 可用性要件

| 項目 | 要件 |
|---|---|
| 稼働時間 | 平日 8:00〜22:00 |
| DB障害時 | エラーメッセージ表示・再試行ガイダンス |
| 接続障害時 | 接続失敗の詳細診断（ConnectionFailure / AuthenticationFailure / TooManyConnections） |

### 4.4 保守性要件

| 項目 | 要件 |
|---|---|
| アーキテクチャ | 3層構造（DataAccess / Business / UI）の分離 |
| デザインパターン | MVP (Passive View) パターンによるテスト容易性の確保 |
| 命名規則 | ディレクトリ名＝名前空間名の統一 |
| テスト | MSTest v4 による単体テスト・Respawnによる統合テスト |
| ログ | ErrorOccurredイベントによるエラーログ機構（将来拡張対応） |

### 4.5 移行要件

| 項目 | 要件 |
|---|---|
| データ移行 | 421テーブルのスキーマ変換（Access→PostgreSQL型マッピング） |
| VBAコード移行 | 50〜70%再利用・残りは再実装 |
| Excel連携 | 2,137件のExcel連携をVB.NETで再実装 |

## 8. 制約条件・前提条件

### 8.1 制約条件

- .NET Framework 4.8 を使用する（.NET Core/.NET 5+への移行は現段階では対象外）
- Windows Forms を採用する（WPF/Blazor等は対象外）
- PostgreSQL 13以降を対象とする
- VBAコードの50〜70%を再利用する方針とする

### 8.2 前提条件

- クライアントPCにはWindows 10/11がインストールされていること
- PostgreSQLサーバーへのネットワーク接続が確保されていること
- 開発にはVisual Studio 2019 または 2022 を使用すること
- テスト用データベース（m7_db_test）がローカル環境に構築済みであること
