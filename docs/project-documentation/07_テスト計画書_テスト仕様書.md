| 2 | No | - | - | - | リース非該当 |
| 3 | Yes | Yes（代替権あり） | - | - | リース非該当 |
| 4 | Yes | No（代替権なし） | No | - | リース非該当 |
| 5 | Yes | No（代替権なし） | Yes | No | リース非該当 |

#### TC-DA-002: エラーハンドリング

**【重要注記】**: `QueryExecutionException`、`DatabaseConnectionException`、`DataAccessErrorCode` は現在のLeaseM4BSコードベース（CrudHelper.vb）に存在しない。CrudHelperは標準の `Exception` および `InvalidOperationException` をスローする。下記テストケースは設計上の仕様であり、実際にテストを実装する場合はコードベースに合わせた例外クラス・エラーコードを使用すること。

| # | テストケース | 操作 | 期待例外 | 期待ErrorCode |
|---|---|---|---|---|
| 1 | 一意制約違反 | 同一キーで2回INSERT | QueryExecutionException | UniqueViolation(23505) |
| 2 | NOT NULL違反 | 必須カラムにNULL | QueryExecutionException | NotNullViolation(23502) |
| 3 | テーブル不在 | 存在しないテーブルSELECT | QueryExecutionException | UndefinedTable(42001) |
| 4 | 構文エラー | 不正SQL | QueryExecutionException | SyntaxError(42601) |
| 5 | 接続失敗 | 不正な接続文字列 | DatabaseConnectionException | ConnectionFailure(8001) |
| 6 | 認証失敗 | 不正なパスワード | DatabaseConnectionException | AuthenticationFailure(28000) |
