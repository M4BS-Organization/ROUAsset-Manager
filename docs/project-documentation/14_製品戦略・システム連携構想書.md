## 1. 製品概要 — 新リース会計基準対応パッケージ

ASBJ第34号（リースに関する会計基準）および第33号（適用指針）（2024年9月公表）を内包した**独立型会計パッケージ**である。既存システムの有無を問わず、これ1つで新基準の法的義務を完遂可能とする。

### 3つの特徴

1. **単体動作**: 既存のリース管理システムや固定資産管理システムがなくても、本パッケージ単独で新リース会計基準への対応が可能
2. **新基準計算エンジン**: ASBJ第34号/第33号に完全準拠した使用権資産・リース負債の計算エンジンを搭載
3. **究極の拡張性**: M4BS（リース管理）/M7（固定資産管理）との連携により、入力工数を最大90%削減可能

---

## 2. 新基準が求める変化

### 旧基準と新基準の比較

| 項目 | 旧基準 | 新基準 |
|---|---|---|
| 対象範囲 | リース管理のみ | 使用権資産（資産計上）+ リース負債（負債計上）の両建て計算 |
| 会計処理 | オフバランス（注記のみ） | オンバランス（B/S計上必須） |

### 新基準で追加される計算要素

- **リース負債算定**: リース料総額の現在価値割引計算
- **資産加算**: 資産除去債務（ARO）、初期直接費用等の使用権資産への加算
- **期間再定義**: 延長オプション・解約オプションの行使可能性評価に基づくリース期間の決定

## 3. 3つの運用モデル

### 3.1 単体運用モデル

**ターゲット**: 既存のリース管理ソフト・固定資産管理ソフトを持たない顧客

#### ワークフロー

```
契約情報直接登録 → 独自マスタ設定 → ARO見積手入力 → 仕訳データ出力
```

#### 4つの主要機能

| 機能 | 内容 |
|---|---|
| 契約管理台帳 | リース契約情報の一元管理 |
| マスタ管理 | 勘定科目・部門・取引先等の独自マスタ |
| 新基準計算 | 使用権資産・リース負債の自動計算 |
| 除去費用集約 | ARO（資産除去債務）の見積・集約管理 |

**メリット**: 既存システムの全面リプレースに伴う膨大なコストを回避し、新基準対応に特化した最小限の投資で法的義務を履行可能。

### 3.2 M4BS連携モデル

**ターゲット**: 既存のリースM4BS（Access）を利用中の顧客

#### アーキテクチャ

```
┌──────────────────┐    ┌──────────────────┐    ┌──────────────────┐
│  リース管理M4BS   │    │  共通基盤CTB     │    │  新基準パッケージ  │
│  (既存Access)    │───→│  (自動変換・     │───→│  (会計計算専用)   │
│                  │    │   マスタ同期)    │    │                  │
│  契約情報供給     │    │                  │    │  仕訳データ出力   │
└──────────────────┘    └──────────────────┘    └──────────────────┘
```

M4BSが契約情報を供給し、新パッケージは会計計算のみを担当する。二重入力を完全に排除する。

#### データ統合マッピング

| M4BSデータ | 新基準パッケージでの利用 |
|---|---|
| 契約期間/支払回数 | 解約不能期間から会計リース期間を決定 |
| 月額リース料 | 利息・元本の自動分離 |
| 追加借入利子率 | 現在価値割引計算 |
| 物件/組織情報 | 減価償却負担の配賦 |

### 3.3 フル統合モデル（M4BS・M7・ARO連携）

**ターゲット**: M4BSおよびシサンM7の両方を利用中の顧客

#### 3データソースがCTBハブに集約

```
┌──────────────────┐
│  リースM4BS       │───┐
│  契約/支払/利率   │   │
└──────────────────┘   │   ┌──────────────────┐    ┌──────────────────┐
                       ├──→│  CTBハブ          │───→│  新基準パッケージ  │
┌──────────────────┐   │   │  (統合・変換)     │    │  (会計計算)       │
│  シサンM7         │───┤   └──────────────────┘    └──────┬───────────┘
│  組織/勘定/配賦   │   │                                   │
└──────────────────┘   │                                   │
                       │                           ┌──────┴───────────┐
┌──────────────────┐   │                           │  シサンM7         │
│  ARO             │───┘                           │  (データ還流)     │
│  除去費用算定基礎 │                               └──────────────────┘
└──────────────────┘
```

| データソース | 提供データ |
|---|---|
| リースM4BS | 契約/支払/利率データ |
| シサンM7 | 組織/勘定/配賦マスタ |
| ARO | 除去費用算定基礎 |

#### 運用メリット

- **現場負担ゼロ**: 既存システムからの自動データ連携により追加入力不要
- **完全データ同期**: CTBハブを介した3システム間のリアルタイムデータ同期
- **最速決算/報告**: 自動計算・自動仕訳により決算作業を大幅短縮

## 4. システム間役割分担

### 4.1 M4BSと新製品の役割分担

| 領域 | M4BS（日常の支払管理） | 新製品（会計・決算） |
|---|---|---|
| 主要機能 | 月次リース料管理 | 現在価値計算 |
| | 振込データ作成 | 使用権資産認識/減価償却 |
| | 再リース手続 | 財務指標シミュレーション |
| | 契約台帳管理 | 会計/税務差異管理 |

### 4.2 新製品とシサンM7の役割分担

| 役割 | 新製品 = 「電卓」 | シサンM7 = 「金庫」 |
|---|---|---|
| 性格 | 正確な決算のための計算専門家 | 資産を守り統合報告するゼネラリスト |
| 主な責務 | リース負債・使用権資産の計算 | 全資産の統合管理・報告 |

#### データ還流

新製品が計算した使用権資産データがシサンM7に自動書き戻しされる。

```
新基準パッケージ ──[使用権資産データ]──→ シサンM7（固定資産台帳に統合）
```

**「一度入力すれば、完璧に管理」**の原則に基づき、データの二重管理を排除する。

## 5. M7へのデータ還流（一元管理モデル）

### 3つのシナリオ

#### シナリオ1: 統合資産台帳

自社保有資産と使用権資産を統合した構成表を出力する。M7の資産台帳に使用権資産が自動登録され、全資産を一元的に把握可能。

#### シナリオ2: 現物棚卸

リース物件にM7資産番号を付与し、バーコードによる棚卸を実施する。リース物件も自社保有資産と同一の管理水準で現物管理が可能。

#### シナリオ3: 償却資産税申告

所有権移転型ROU（使用権）資産の地方税（償却資産税）申告に対応する。M7の申告機能を活用し、使用権資産の税務申告を自動化。

### 3つのメリット

| メリット | 内容 |
|---|---|
| 報告簡素化 | 自社保有+リースの統合レポートを自動生成 |
| 管理水準統一 | リース物件にも自社保有と同一の管理ルールを適用 |
| 自動連動 | 新基準パッケージの計算結果がM7に自動反映 |

## 6. 管理対象範囲

不動産だけでなく、**全種類のリース資産**を管理対象とする。

### 対象資産カテゴリ

| カテゴリ | 具体例 |
|---|---|
| 不動産 | オフィス、倉庫、店舗、社宅 |
| 車両 | 営業車、トラック、フォークリフト |
| OA機器/什器 | コピー機、PCサーバー、各種産業機械 |

CTBの汎用設計により、特定の資産種別に依存しないアーキテクチャとなっている。資産種別の追加は、マスタ設定の変更のみで対応可能。
