## 1. プロジェクト概要

### 1.1 背景

MS Access（.mdb）で構築された業務アプリケーションを、PostgreSQL + VB.NET（Windows Forms）のクライアントサーバー型アプリケーションへ移行するプロジェクトである。本プロジェクトでは以下の2つのサブシステムが関連している。

1. **シサンM7（固定資産管理）**: ms-access-migration リポジトリで開発中。データアクセス層・UI基盤の移行。
2. **LeaseM4BS（リース管理）**: LeaseM4BS-1 リポジトリで開発中。リース契約管理・新リース会計基準対応。

### 1.2 プロジェクト目的

- Access DAOパターンからPostgreSQL + Npgsqlへのデータアクセス移行
- 企業会計基準第34号（ASBJ第34号）に準拠したリース判定・会計処理機能の実装
- マルチユーザー対応のクライアントサーバー型アーキテクチャへの移行

## 2. システム構成

### 2.1 リポジトリ構成

本プロジェクトは2つのリポジトリで構成される。

**リポジトリ1: ms-access-migration（シサンM7移行基盤）**

```
ms-access-migration/
├── analysis/              … 解析資料
├── postgresql/            … SQLスクリプト（421テーブル）
├── forms_analysis/        … VBAフォーム分類
├── vbnet-data-access/     … VB.NETコード
    ├── DataAccess/        … データアクセス層（構築済み）
    ├── Business/          … ビジネス層（新規構築中）
    └── UI/                … UI層（新規構築中）
```

**リポジトリ2: LeaseM4BS-1（リース管理システム）**

```
LeaseM4BS-1/
├── LeaseM4BS/
│   ├── LeaseM4BS.DataAccess/          # データアクセス層
│   │   ├── CrudHelper.vb
│   │   ├── DbConnectionManager.vb
│   │   └── UsageExamples.vb           # 使用例・サンプルコード集（362行）
│   └── LeaseM4BS.Tests/              # 単体テスト
│       └── UserCrudTests.vb
├── LeaseM4BS.TestWinForms/
│   └── LeaseM4BS.TestWinForms/        # WinFormsアプリ本体
│       ├── FrmFlexMenu.vb             # メインメニュー
│       ├── FrmFlexContract.vb         # 契約書一覧
│       ├── FrmLeaseContractMain.vb    # リース契約詳細（5タブ）※Designer.vbなし・コードオンリー
│       ├── FrmAssetDetailEntry.vb     # 資産詳細入力（Designer.vb・.resxあり）
```

### 2.3 接続文字列の優先順位（LeaseM4BS）

1. `App.config` の `connectionStrings["LeaseM4BS"]`
2. 環境変数 `LEASE_M4BS_CONNECTION_STRING`
3. デフォルト値（localhost:5432, lease_m4bs DB, ユーザー名・パスワードを含む）

> **セキュリティ注意**: デフォルト値にはユーザー名とパスワードが平文でハードコードされている（`DbConnectionManager.vb`参照）。本番環境では必ず `App.config` または環境変数で上書きすること。パスワードをgitに含めないよう注意が必要。

### 3.3 会計タブ — ASBJ第34号に基づく自動計算ロジック

| 計算項目 | 根拠条文 | 計算式 |
|---|---|---|
| 会計期間 | 第34号§17 | 基本期間(月) + 更新予想回数 × 更新月数 |
| 賃料総額 | - | 月額賃料 × 基本期間月数 |
| 算定総額 | 第34号§22 | 月額賃料 × 会計期間月数 |
| リース・非リース配分 | 第34号§13 | 算定総額 × リース割合 / (リース割合 + 非リース割合) |
| 現在価値(PV) | 第34号§22 | 月額支払 × (1-(1+r)^-n)/r |
| 使用権資産 | 第34号§27 | PV + 初期直接費用 + 原状回復費用 - リース・インセンティブ |
| リース負債 | 第34号§22 | PV |

### 3.4 返済スケジュールマトリックス項目

- 使用権資産（ROU）: 期首・増加・変更増減・減少・期末
- リース負債（Liab）: 期首・増加・変更増減・減少・期末
- 除去債務（ARO: Asset Retirement Obligation）: 期首・増加・変更増減・減少・期末

## 4. 非機能要件

### 4.1 セキュリティ要件

| 項目 | 対策 |
|---|---|
| SQLインジェクション | パラメータ化クエリの徹底 |
| 動的SQL構築 | ホワイトリスト方式（FlexDataService） |
| 演算子検証 | 許可リスト（=, LIKE, >, <, >=, <=, <>） |
| カラム名エスケープ | PostgreSQL二重引用符 |
| エラーハンドリング | CrudHelperは標準の `Exception` をスロー（詳細メッセージ付き再スロー） |

> **注意**: `DataAccessErrors` 体系（`DataAccessException`継承ツリー・`DataAccessErrorCode`列挙体）はLeaseM4BSには実装されていない。これらはシサンM7（ms-access-migration）側の成果物であり、LeaseM4BSのコードベースには存在しない。

### 4.2 技術的制約

- .NET Framework 4.7.2 / 4.8（.NET Core/.NET 5+ではない）
- Npgsql 6.0.11固定（NuGet packages.config管理）
- FrmLeaseContractMainはコードオンリーUI構築（Designerなし、38000トークン超）
- RecalcAll()が複数タブにわたる再計算を入力イベントのたびに実行
- サンプルデータはハードコーディング（DBとの実CRUDは未接続: LoadSampleData()）
- ヘッダーボタン（登録・修正・変更・更新・満了解約・中途解約・削除）は「この機能は現在実装中です。」メッセージのみ
- ARO計算ロジックはマトリックス列はあるが詳細計算は未実装
