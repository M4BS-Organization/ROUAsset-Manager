## 3. コーディングスタイル

### 3.1 データアクセスパターン

**CrudHelperパターン（推奨）**

```vb
' ○ 正しい使用法
Using helper As New CrudHelper()
    Dim sql As String = "SELECT * FROM tw_lease_contract WHERE contract_no = @no"
    Dim selectParams As New List(Of NpgsqlParameter) From {
        New NpgsqlParameter("@no", contractNo)
    }
    Dim dt As DataTable = helper.GetDataTable(sql, selectParams)
End Using
```

> **注意**: `GetDataTable` の実際のシグネチャは以下のとおり（CrudHelper.vb 参照）:
> ```vb
> Public Function GetDataTable(sql As String, Optional parameters As List(Of NpgsqlParameter) = Nothing) As DataTable
> ```
> 引数型は `Dictionary(Of String, Object)` ではなく `List(Of NpgsqlParameter)` です。

**パラメータ化クエリの徹底（必須）**

```vb
' ○ パラメータ化クエリ（安全）
Dim sql = "SELECT * FROM users WHERE name = @name"
Dim params As New List(Of NpgsqlParameter) From {New NpgsqlParameter("@name", inputValue)}

' × 文字列連結（禁止）
Dim sql = "SELECT * FROM users WHERE name = '" & inputValue & "'"
```

### 3.2 エラーハンドリング

> **重要**: `QueryExecutionException`、`DatabaseConnectionException`、`DataAccessErrorCode` は **LeaseM4BS のコードベースに存在しません**。これらはシサンM7（ms-access-migration）側の設計パターンです。LeaseM4BS の `CrudHelper.vb` は標準の `Exception` をスローするため、以下の `Catch ex As Exception` パターンを使用してください。

```vb
' ○ LeaseM4BS での正しいエラーハンドリングパターン
Try
    helper.ExecuteNonQuery(sql, parameters)
Catch ex As Exception
    ' PostgreSQL エラーコードが必要な場合は InnerException から NpgsqlException を取得
    Dim pgEx As Npgsql.NpgsqlException = TryCast(ex.InnerException, Npgsql.NpgsqlException)
    If pgEx IsNot Nothing Then
        Select Case pgEx.SqlState
            Case "23505"  ' unique_violation
                MessageBox.Show("このデータは既に登録されています",
                    "重複エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning)
            Case "23502"  ' not_null_violation
                MessageBox.Show("必須項目が入力されていません",
                    "入力エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning)
            Case Else
                MessageBox.Show($"データベースエラー: {ex.Message}",
                    "エラー", MessageBoxButtons.OK, MessageBoxIcon.Error)
        End Select
    Else
        MessageBox.Show($"予期しないエラー: {ex.Message}",
            "エラー", MessageBoxButtons.OK, MessageBoxIcon.Error)
    End If
End Try
```

### 3.3 VB.NET Binary If の制約

```vb
' ○ Nullable型 → Binary If 使用可
parameters.Add("@nullable_int", If(entity.NullableInt, DBNull.Value))

' ○ 参照型（String）→ Binary If 使用可
parameters.Add("@name", If(entity.Name, DBNull.Value))

' × 非Nullable値型（Integer, Boolean）→ Binary If 使用不可
' parameters.Add("@bpnid", If(entity.Bpnid, DBNull.Value))  ← コンパイルエラー

' ○ 非Nullable値型は直接代入
parameters.Add("@bpnid", entity.Bpnid)
parameters.Add("@calc_k", entity.CalcK)
```

### 3.4 状態管理

> **注意**: `FrmLeaseContractMain` では `Form_Load` イベントは使用しておらず、`Sub New` コンストラクタで初期化処理（`InitTabContract()`、`InitTabInitialCost()` 等）を行っています。`_isLoaded` フラグは `Sub New` 末尾で `True` に設定し、コンストラクタ実行中のイベント発火を防止します。

```vb
' _isLoaded フラグによる初期化制御
Private _isLoaded As Boolean = False

' FrmLeaseContractMain の場合: Sub New コンストラクタで初期化
Public Sub New()
    InitializeComponent()
    InitTabContract()
    ' ... 各タブ初期化 ...
    _isLoaded = True  ' ← Sub New 末尾で設定
End Sub

Private Sub ComboBox_SelectedIndexChanged(sender As Object, e As EventArgs)
    If Not _isLoaded Then Return  ' 初期化中のイベント発火を防止
    RecalcAll()
End Sub
```

### 3.5 自動採番パターン

' 静的カウンタによる採番
Private Shared _contractCounter As Integer = 0

    txtContractNo.Text = $"LC-{DateTime.Now.Year}-{_contractCounter:D4}"

## 5. テストコード規約

### 5.1 MSTest v4 アサーション（推奨メソッド）

> **注意**: LeaseM4BS は現在 **MSTest v2.2.8** を使用しています。以下の推奨メソッド（`Assert.IsGreaterThan`、`Assert.HasCount`、`Assert.IsEmpty` 等）は MSTest v4 で追加されたものであり、MSTest v2.2.8 では利用できません。MSTest v4 へのアップグレード後に利用可能となります。現行バージョンでは「非推奨」列のメソッドを使用してください。

| 目的 | MSTest v4 推奨メソッド（※未導入） | MSTest v2.2.8 互換（現行使用） |
|---|---|---|
| 数値比較 | Assert.IsGreaterThan(value, 0L) | Assert.IsTrue(value > 0) |
| 文字列含有 | StringAssert.Contains(str, "Host=") | Assert.IsTrue(str.Contains("Host=")) |
| コレクション件数 | Assert.HasCount(3, collection) | Assert.AreEqual(3, collection.Count) |
| 空コレクション | Assert.IsEmpty(collection) | Assert.AreEqual(0, collection.Count) |
| Boolean | Assert.IsTrue(value) / Assert.IsFalse(value) | Assert.AreEqual(True, value) |

**注意**: Assert.HasCountの引数順は `(expectedCount, collection)` の順。
