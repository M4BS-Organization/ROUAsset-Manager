## 2. インストール・セットアップ

### 2.1 前提条件

| 項目 | 要件 |
|---|---|
| OS | Windows 10/11 |
| .NET Framework | 4.7.2以上（LeaseM4BS）/ 4.8以上（シサンM7） |
| PostgreSQL | 13以上（ローカルまたはRDS） |
| ディスク容量 | アプリ: 50MB以上 / DB: 1GB以上推奨 |
| メモリ | 4GB以上推奨 |

### 2.2 PostgreSQLセットアップ

```bash
# PostgreSQLのインストール後
# データベース作成（シサンM7用）
createdb -U postgres m7_db
createdb -U postgres m7_db_test

# LeaseM4BS 用データベース作成
createdb -U postgres lease_m4bs        # 本番・開発共通
createdb -U postgres lease_m4bs_dev    # 開発環境専用（App.configの LeaseM4BS_Dev）
createdb -U postgres lease_m4bs_test   # テスト環境専用（App.configの LeaseM4BS_Test）

# テーブル作成（シサンM7 — 421テーブル）
psql -U postgres -d m7_db -f postgresql/create_all_tables.sql

# テスト用テーブル作成
psql -U postgres -d m7_db_test -f postgresql/create_all_tables.sql
```

### 2.3 接続文字列設定

**シサンM7 (App.config)**

```xml
<connectionStrings>
  <add name="M7Database"
       connectionString="Host=localhost;Port=5432;Database=m7_db;Username=m7user;Password=xxxxx;Pooling=true;MinPoolSize=1;MaxPoolSize=20"
       providerName="Npgsql" />
</connectionStrings>
```

**LeaseM4BS (App.config)** — 実際の設定例（LeaseM4BS.TestWinForms/App.config より）

```xml
<connectionStrings>
  <!-- 本番・開発接続文字列 -->
  <add name="LeaseM4BS"
       connectionString="Host=localhost;Port=5432;Database=lease_m4bs;Username=lease_m4bs_user;Password=xxxxxxxx;Pooling=true;Minimum Pool Size=1;Maximum Pool Size=20;Connection Lifetime=0"
       providerName="Npgsql" />

  <!-- 開発環境用（オプション） -->
  <add name="LeaseM4BS_Dev"
       connectionString="Host=localhost;Port=5432;Database=lease_m4bs_dev;Username=lease_m4bs_user;Password=xxxxxxxx"
       providerName="Npgsql" />

  <!-- テスト環境用（オプション） -->
  <add name="LeaseM4BS_Test"
       connectionString="Host=localhost;Port=5432;Database=lease_m4bs_test;Username=lease_m4bs_user;Password=xxxxxxxx"
       providerName="Npgsql" />
</connectionStrings>
```

> **セキュリティ警告**: `DbConnectionManager.vb` のデフォルトフォールバックには開発用パスワードがハードコードされています。**本番環境では必ず App.config または環境変数で接続文字列を設定し、デフォルト値のまま運用しないでください。**

**環境変数による上書き（LeaseM4BS）**

```bash
SET LEASE_M4BS_CONNECTION_STRING=Host=prodserver;Port=5432;Database=lease_m4bs;Username=xxx;Password=xxx
```

接続文字列の優先順位: App.config → 環境変数 → デフォルト値（localhost:5432、ユーザー名・パスワードを含む）

## 3. 起動・停止手順

### 3.1 アプリケーション起動

1. デスクトップのショートカットまたはexeファイルをダブルクリック
2. FrmFlexMenu（メインメニュー）が表示される
3. 起動後、コンストラクタ内で `SwitchContent(btnContract)` が呼ばれ、**デフォルトで契約書(フレックス)画面が表示される**
4. メニューボタンから各機能にアクセス

### 3.2 接続確認

起動時にPostgreSQL接続テストが実行される。失敗時は以下を確認する。

| エラーコード | メッセージ | 対処 |
|---|---|---|
| ConnectionFailure(8001) | データベースサーバーに接続できません | PostgreSQLが起動しているか確認。ネットワーク接続を確認 |
| AuthenticationFailure(28000) | 認証に失敗しました | App.configのユーザー名/パスワードを確認 |
| TooManyConnections(53300) | 接続数が上限に達しています | 他のセッションを終了するか、max_connectionsを増やす |
| ConfigurationError(90001) | 設定エラー | App.configのconnectionStringsセクションを確認 |

## 4. 日常運用

### 4.1 バックアップ

```bash
# 日次バックアップ（推奨: 毎日22:00実行）
pg_dump -U postgres -Fc m7_db > /backup/m7_db_$(date +%Y%m%d).dump
pg_dump -U postgres -Fc lease_m4bs > /backup/lease_m4bs_$(date +%Y%m%d).dump

# 週次フルバックアップ
pg_dumpall -U postgres > /backup/full_$(date +%Y%m%d).sql
```

### 4.2 リストア

```bash
# データベース単体リストア
pg_restore -U postgres -d m7_db -c /backup/m7_db_20260309.dump

# 完全リストア
dropdb -U postgres m7_db
createdb -U postgres m7_db
pg_restore -U postgres -d m7_db /backup/m7_db_20260309.dump
```

### 4.3 DB接続状態の確認

```sql
-- アクティブ接続の確認
SELECT pid, usename, datname, state, query_start, query
FROM pg_stat_activity
WHERE datname IN ('m7_db', 'lease_m4bs')
ORDER BY query_start;

-- テーブル件数の確認
SELECT schemaname, relname, n_live_tup
FROM pg_stat_user_tables
ORDER BY n_live_tup DESC
LIMIT 20;
```

### 4.4 ログ確認

```bash
# PostgreSQLログ
tail -f /var/log/postgresql/postgresql-13-main.log

# アプリケーションエラーログ
# DbConnectionManager.WriteError() によりアプリケーション実行ディレクトリの error.log に出力される（相対パス）
# 例: C:\Program Files\LeaseM4BS\error.log （実行ディレクトリに依存）
```

> **注意**: `ErrorOccurred` イベントは現在のコードベースには実装されていません（**将来実装予定**）。現状のエラーログは `DbConnectionManager.WriteError()` による静的メソッド経由でのみ出力されます。

## 5. 障害対応

### 5.1 障害判定フロー

```
画面が表示されない / エラーメッセージが出る
  ↓
接続エラーか？
  ├── Yes → 「5.2 DB接続障害」へ
  └── No → データエラーか？
              ├── Yes → 「5.3 データ障害」へ
              └── No → 「5.4 アプリケーション障害」へ
```

### 5.2 DB接続障害

| 症状 | 原因候補 | 対処 |
|---|---|---|
| 接続タイムアウト | PostgreSQL停止 | `sudo systemctl start postgresql` で起動 |
| 認証エラー | パスワード変更 | App.configの接続文字列を更新 |
| 接続数上限 | 同時接続過多 | `pg_stat_activity`で確認、不要セッションをKILL |
| ネットワークエラー | VPN切断等 | ネットワーク接続を確認 |

```sql
-- 不要セッションの強制終了
SELECT pg_terminate_backend(pid)
FROM pg_stat_activity
WHERE datname = 'm7_db' AND state = 'idle' AND query_start < NOW() - INTERVAL '30 minutes';
```

### 5.3 データ障害

| 症状 | 原因候補 | 対処 |
|---|---|---|
| 一意制約違反(23505) | 重複データ登録 | 既存データを確認・重複排除 |
| 外部キー違反(23503) | 参照先データ不在 | マスタデータの存在を確認 |
| デッドロック(40P01) | 同時更新の競合 | 自動リトライ。頻発時はトランザクション設計を見直し |

### 5.4 アプリケーション障害

| 症状 | 対処 |
|---|---|
| 画面がフリーズ | RecalcAll()の実行中の可能性。しばらく待つか、タスクマネージャーで強制終了 |
| メモリ不足 | 大量データ表示時に発生しうる。アプリ再起動 |
| 予期しない例外 | エラーダイアログの内容を記録し、開発チームに報告 |

## 6. 監視項目

### 6.1 定期確認項目

| 項目 | 頻度 | 方法 | 閾値 |
|---|---|---|---|
| DB接続数 | 日次 | pg_stat_activity | 最大接続数の80%で警告 |
| テーブルサイズ | 週次 | pg_total_relation_size | 急激な増加がないか確認 |
| バキューム状態 | 週次 | pg_stat_user_tables.last_vacuum | 1週間以上未実行で警告 |
| バックアップ | 日次 | ファイル存在確認 | 前日分が存在すること |
| ディスク使用率 | 日次 | df -h | 80%で警告、90%で対応 |

### 6.2 将来のログ・監視機能

コードベースにはErrorOccurredイベントが実装済みであり、以下の監視情報が取得可能な設計になっている。

- トランザクション実行時間（Duration）
- 影響行数（AffectedRows）
- 相関ID（CorrelationId）— 将来の分散トレーシング用
- 操作名（OperationName）
- SQL文とパラメータ

## 7. 定期メンテナンス

### 7.1 PostgreSQLメンテナンス

```bash
# VACUUM ANALYZE（週次推奨）
vacuumdb -U postgres --analyze m7_db
vacuumdb -U postgres --analyze lease_m4bs

# 統計情報の更新
psql -U postgres -d m7_db -c "ANALYZE;"

# インデックスの再構築（月次推奨）
psql -U postgres -d m7_db -c "REINDEX DATABASE m7_db;"
```

### 7.2 アプリケーション更新手順

1. 現行バージョンのバックアップ（exeフォルダを丸ごとコピー）
2. ユーザーに終了を依頼
3. 新バージョンのexeを配布
4. App.configの接続文字列が変更されていないことを確認
5. 起動テスト実施
6. ユーザーに使用再開を通知
