# 要件定義: Issue #8 f_flx_KLSRYO 計算ロジック

## 機能要件

Access版 pc_SHRI_KLSRYO の計算エンジンをVB.NETに忠実に移植する。

### 入力パラメータ (JOKEN画面から)
1. 期間FROM (Date)
2. 期間TO (Date)
3. 集計対象: 1=リース料, 2=保守料, 3=全部
4. 計上タイミング: 1=締日ベース, 2=支払日ベース
5. 明細単位: 1=物件単位, 2=配賦単位

### 出力項目 (DataGridView列)
- 契約情報: 物件No, 再回, 配No, 契区, 計上区分, リース区分, 契約番号, 支払先, 物件名
- 部署情報: 管理部署, 費用負担部署, 費用区分
- 日付: 開始日, 終了日, 中途解約日
- **行区 (REC_KBN_STR)**: 定額/変額/付随費用
- **法令区分**: 新法/旧法 (施行日比較)
- **取引区分**: 費用計上/資産計上
- **請求月**: 集計期間の月表示
- **回数済/総**: 済回数/総回数
- 金額: 現金購入価額_物件, 総支払額
- **計算結果**: 前期末残高, 当期額, 月別内訳(G01-G12), 期末残高, 内1年内〜5年超, 期中増加

### 計算ロジック要件
1. キャッシュスケジュール生成 (支払予定配列)
2. スケジュールから期間集計 (前期以前/当期/翌期以降)
3. 翌期以降の年度内訳 (1年内/2年内/.../5年超)
4. 当期の月別内訳 (最大12ヶ月)
5. 自動更新契約の特殊処理

## 非機能要件
- MSBuild でビルドが通ること
- 既存の CrudHelper, FormHelper パターンに準拠
